# 列表、树和网格
## 22.1ListView控件
ListView类是一个特殊的列表类，它是专门针对显示相同数据的不同视图而设计的。从技术的角度来看。ListView类的View属性指向继承自ViewBase类的任意类的实例。ViewBase类是一个将两个样式捆绑在一起的包。其中的一个样式应用到ListView控件（并通过DefaultStyleKey属性加以引用），而另一个样式应用到ListView控件中的项（并通过ItemContainerDefaultStyleKey属性加以应用）。DefaultStyleKey和ItemContainerDefaultStyleKey属性实际上没提供树木样式，只是返回指向样式的ResourceStyle对象。
ListView控件的优点：
- 可重用视图：ListView控件将所有特定于视图的细节分离到一个对象中，这样便于创建不依赖于数据并且可用于多个列表的视图。
- 多视图：将ListView控件和View对象分离开来，还使得对同一列表切换多个视图变得更容易。可通过动态改变模板和样式来构建相同的特性，但只使用一个封装了所有视图细节的对向更容易。
- 更好的组织：视图对象封装了两个样式：一个用于ListView根控件；另一个用于列表中的单个项。应为这些样式封装在一起，所以很明显这两部风是相关联的，可共享特定的细节并相互依赖。

#### 20.1.1.1单元格模板
为在单元格中显示数据，GridViewColumn.DisplayMemberBinding属性不是唯一的选择。另一个选择是使用CellTemplate属性，该属性使用模板。但是改方法具有唯一的限制，数据模板需要明确绑定到希望显示的属性，因而每列的模板无法通用。
示例：
```XAML

<GridViewColumn Header="Description" Width="300">
  <GridViewColumn.CellTemplate>
    <DataTemplate>
      <TextBlock Text="{Binding Path=Description}" TextWrapping="Wrap"/>
    </DataTemplate>
  </GridViewColumn.CellTemplate>
</GridViewColumn>
```
==同时：可使用GridViewColumn.HeaderTemplate属性定义该列头的模板，也可以使用GridColumnContainerStyle属性提供样式。同时可以为其配备第20章所学的样式选择器和模板选择器==

### 22.1.2创建自定义视图
视图通过重写两个受保护的属性进行工作:DefaultStyleKey和ItemContainerDefaultStyleKey。这两个属性都返回名为ResourceKey的特殊对象，该对象指向在XAML中定义的样式。DefaultStyleKey属性指向将被用于配置整个ListView控件的样式，而ItemContainer.DefaultStyleKey属性指向将被用于配置ListView控件中每个LiteViewItem元素的样式。其自定义的目的在于实现列表动态地改变视图。
#### 22.1.2.1视图类
创建自定义视图类时，该类必须继承自ViewBase类。该TileView类应用了样式：TileView（用于ListView)和TileViewItem（用于ListViewItem)。此外，TileView类还定义了ItemTampLate属性，使得使用TileView类的开发人员能够提供正确的数据模板。然后这个模板被插入到每个ListViewItem的内部，用于创建平铺显示的内容。
示例：
```Csharp

public class TileView:ViewBase
{
    private DataTemplate _itemTemplate;
    public DataTemplate ItemTemplate
    {
        get { return _itemTemplate; }
        set { _itemTemplate = value; }
    }
//ComponentResourceKey:封装了两部分信息：拥有样式的类的类型，以及表示资源的描述性的ResourceId字符串
    protected override object DefaultStyleKey
    {
        get { return new ComponentResourceKey(GetType(), "TileView"); }
    }

    protected override object ItemContainerDefaultStyleKey
    {
        get { return new ComponentResourceKey(GetType(), "TileViewItem"); }
    }
}
```

#### 22.1.2.2视图样式
WPF需要能够找到希望使用的样式。确保能够自动获取的样式的技巧是创建一个名为generic.xaml资源字典。这个资源字典必须被放在项目文件夹下Themes子文件中。
示例：
```XAML

<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:local="clr-namespace:WPF第二十二章学习文件.Models"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Style x:Key="{ComponentResourceKey TypeInTargetAssembly={x:Type local:TileView},ResourceId=TileView}" TargetType="{x:Type ListView}"
                           BasedOn="{StaticResource {x:Type ListBox}}"> 
        <Setter Property="BorderBrush" Value="red"/>
        <Setter Property="BorderThickness" Value="1"/>
        <Setter Property="Grid.IsSharedSizeScope" Value="True"/>
        <Setter Property="ItemsPanel">
            <Setter.Value>
                <ItemsPanelTemplate>
                    <WrapPanel Width="{Binding (FrameworkElement.ActualWidth),RelativeSource={RelativeSource AncestorType=ScrollContentPresenter}}"/>
                </ItemsPanelTemplate>
            </Setter.Value>
        </Setter>
    </Style>

    <Style x:Key="{ComponentResourceKey TypeInTargetAssembly={x:Type local:TileView},ResourceId=TileViewItem}" TargetType="{x:Type ListViewItem}"
           BasedOn="{StaticResource {x:Type ListBoxItem}}">
        <Setter Property="Padding" Value="3"/>
        <Setter Property="BorderBrush" Value="Black"/>
        <Setter Property="BorderThickness" Value="1"/>
        <Setter Property="HorizontalContentAlignment" Value="Center"/>
        <Setter Property="ContentTemplate" Value="{Binding Path=View.ItemTemplate,RelativeSource={RelativeSource Mode=FindAncestor,AncestorType={x:Type ListView}}}"/>
    </Style>
</ResourceDictionary>
```

## 22.2TreeView控件
TreeView控件本质上是驻留TreeViewItem对象的特殊ItemControl控件。但与ListViewItem对下个不同，TreeViewItem对象不是内容控件。相反，每个TreeViewItem对象都是单独的ItemsControl控件，可以包含多个TreeViewItem对象。
示例：
```XAML
<StackPanel>
    <TreeView>
        <TreeViewItem Header="Fruit">
            <TreeViewItem Header="Orange"/>
            <TreeViewItem Header="Banana"/>
            <TreeViewItem Header="Grapefruit"/>
        </TreeViewItem>
        <TreeViewItem Header="Vegetables">
            <TreeViewItem Header="Aubergine"/>
            <TreeViewItem Header="Squash"/>
            <TreeViewItem Header="Spinach"/>
        </TreeViewItem>
    </TreeView>
</StackPanel>
```
可以为TreeView控件添加任何元素，包扣按钮、面板以及图像。如果希望显示非文本内容，可以使用TreeViewItem分装器，并通过TreeViewItem.Header内容提供内容。

### 22.2.1创建数据绑定到TreeView控件
使用数据填充TreeView控件非常简单——与任意ItemsCotorl控件一样，只需要设置ItemSource属性。同时在创建模板时，使用HierarchicalDataTemplate对象，它能够封装第二个模板。然后HierarchicalDataTemplate对象就可以从第一层数据中提项的数据集合，并将提供给第二层的模板。
示例：
```XAML
 <TreeView >
     <TreeView.ItemTemplate>
         <HierarchicalDataTemplate ItemsSource="{Binding Path=Products}">
             <TextBlock Text="{Binding Path=CategoryName}"/>
             <HierarchicalDataTemplate.ItemTemplate>
                 <DataTemplate>
                     <TextBlock Text="{Binding Path=ModelName}"/>
                 </DataTemplate>
             </HierarchicalDataTemplate.ItemTemplate>
         </HierarchicalDataTemplate>
     </TreeView.ItemTemplate>
 </TreeView>
```
### 22.2.3即时创建节点
创建即时节点时，当展开每个ThreeViewItem对象时会引发Expanded事件，并且当关闭时会引发Collapsed事件。可通过处理这两个事件即时填充丢失的节点或者丢弃不再需要的节点。这种技术叫做即时创建节点。

### 22.3.1改变列的尺寸与重新安排
当显示自动生成的列时，DataGrid控件尝试根据DataGrid.ColumnWidth属性智能地改变每列的宽度。其中SizeToHeader使列宽随着表头确定，SizeToCells使列宽随单元格中最长行而定，Auto使列宽自动而变，使用其中列或者单元格最大的值。同时也可用 “*”来设定列宽比例。
示例：
```XAML
<DataGridTextColumn Header="用户状态" Width="SizeToCells" Binding="{Binding Status}"/>
```

### 22.3.2定义列

DataGrid控件支持几种类型的列，通过继承自DataGridColumn的不同类表示这些列：
- DataGridTextColumn：这种列对大部分数据类型是标准的选择。值转换成文本，并在TextBlock元素中显示，当编辑行时，TextBlock元素被替换为标准的文本框。
-  DataGridCheckBoxColumn:这种类显示复选框。为Boolean（或可空Boolean)值自动使用这种类型。通常，复选框时只读的；当编辑行时，会编程普通的复选框。
- DataGridHyperlinkColumn:这种列显示可单击的链接。如果结合使用URI(通常是外部的Web战站点)。
- DataGridComboBox:最初这种列看起来与DataGridTextCloumn类似，但在编辑模式下这种列会变成下拉的ComboBox控件。当希望将编辑限制于允许的少部分值时，这种列时很好的选择。
- DataGridTemplateColumn:这种列是到目前为止功能最强大的选择。这种列允许为显示列值定义数据模板，具有在列表控件中使用模板是所有具有的所有灵活性和功能。

1.DataGridCheckBoxColumn
与DataGridTextColumn一样，Binding属性提取数据——对于这种情况，是用于设置内部CheckBox元素的IsChecked属性的true或者False值。DataGridCheckBoxColumn还添加了Content属性，通过该属性可在复选框旁边显示可选的内容。最后，DataGridCheckBoxColumn提供了IsThreeState属性，该属性确定复选框是否支持未定状态以及更明显的选中和未选中状态。如果使用DataGridCheckBoxColumn显示来自可空Bolean值的信息，可将IsThreeState属性设置未True。这样用户可通过单击切换到未定状态（显示具有清淡阴影的复选框）,以便未绑定的值返回Null。

2.DataGridHyperlinkColumn
DataGridHyperlinkColumn允许显示的文本值包含单个URL。==注意：仅当支持导航时间的容器（如Frame或者Nacigatin Window)中放置DataGrid控件时，这种导航才会奏效。==
