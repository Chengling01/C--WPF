# 几何图形和图画
## 13.1路径几何图形
Path类能够包含任何简单形状、多组形状以及更复杂的要素。同时Path类提供了Data属性，该属性接受一个Geometry对象，该对象定义路径包含的一个或多个图形。可通过一下七个子类创建。

|         名称         |                                           说明                                           |
| :----------------: | :------------------------------------------------------------------------------------: |
|   LineGemometry    |                                  代表直线，该几何图形相当于Line形状                                   |
| RectangleGemometry |                           代表矩形（可以具有圆形拐角），该几何图形相当于Rectangle形状                           |
|  EllipseGeometry   |                                  代表椭圆，该图形相当于Ellipse形状                                  |
|   GeometryGroup    |                  为单个路径添加任意多个Geometry对象，使用EvenOdd或NonZero填充规则来确定要填充的区域                  |
| CombinedGeomentry  |                      将两个几何图形合并为一个形状。可使用CombineMode属性选择如何组合两个几何图形                       |
|    PathGeometry    |                        代表更复杂的有弧线、曲线以及直线构成的图形，并且既可以是闭合的，也可以是不闭合的                        |
|   SteamGeometry    | 相当于PathGeometry的只读的轻量级类。StreamGometry图形可以节约内存，因为它不再内存中同时保存路径的所有单个分段。并且这些类入刑一旦被创建就不能再修改 |
## 13.1.1直线、矩形和椭圆图形

- 矩形的使用示例：
```XAML

<Rectangle Fill="Yellow" Stroke="Blue" StrokeThickness="1" Width="100"
           Height="50"/>
<Path Fill="Yellow" Stroke="Blue" HorizontalAlignment="Center" Margin="0,10">
    <Path.Data>
        <RectangleGeometry Rect="0,0 150,50"/>
    </Path.Data>
</Path>
```

- 直线的示例：
```XAML

<Line Stroke="Blue" X1="0" Y1="0" X2="10" Y2="100"/>
<Path Fill="Yellow" Stroke="Blue">
    <Path.Data>
        <LineGeometry StartPoint="0,0" EndPoint="10,100"/>
    </Path.Data>
</Path>
```

- 椭圆的示例：
```XAML

 <Ellipse Fill="Yellow" Stroke="Blue" Width="100" Height="50"/>
 <Path Fill="Yellow" Stroke="Blue">
     <Path.Data>
         <EllipseGeometry RadiusX="50" RadiusY="35" Center="250,25"/>
     </Path.Data>
 </Path>
```

## 13.1.2使用GeometryGroup组合形状
组合图形最简单的方法是使用GeometryGroup队形，该对象在内部嵌套其他Geometry类的派生类对象。
- 示例：
```XAML
<Path Fill="Yellow" Stroke="Blue" Margin="5" >
    <Path.Data>
        <GeometryGroup>
            <RectangleGeometry Rect="0,0 100,100"/>
            <EllipseGeometry Center="150,50" RadiusX="35" RadiusY="25"/>
        </GeometryGroup>
    </Path.Data>
</Path>
```

几何图形的另一个优点的可几个独立的Path元素中重用相同的几何图形。只需要在Resources集合中定义几何图形，并通过静态资源或者动态资源进行引用即可。
- 示例：
```XAML
 <Window.Resources>
     <GeometryGroup x:Key="Geometry">
         <RectangleGeometry Rect="0,0 100,100"/>
         <EllipseGeometry Center="150,50" RadiusX="35" RadiusY="25"/>
     </GeometryGroup>
 </Window.Resources>
 
  <Path Fill="Red" Stroke="Blue" Data="{StaticResource Geometry}"/>
```

当组合图形出现交叉或者重叠的情况时，GeometryGroup将十分有趣。这时可以使用其FillRule属性（该属性可设为：EvenOdd或者Nonzero)来控制那些图形将被填充。
- 示例：
```XAML
<Path Fill="Green" Stroke="Red" Margin="100">
    <Path.Data>
        <GeometryGroup FillRule="EvenOdd">
            <RectangleGeometry Rect="0,0 100,100" />
            <EllipseGeometry Center="50,50" RadiusX="35" RadiusY="25"/>
        </GeometryGroup>
    </Path.Data>
</Path>
```

## 13.1.3使用CombinedGeometry融合几何图形
CombinedGeometry类专门用于组合重叠到一起并且部相互包含的形状。与Geometry类不同，CombinedGeometry类只使用两个几何图形，通过Geomentry1和Geomentry属性提供者两个几何图形。同时CombinedGeometry类提供了更强大的GeometryCombineMode属性，其值下表所示：

|    名称     |                 说明                 |
| :-------: | :--------------------------------: |
|   Union   |         创建包含两个几何图形所有区域的形状          |
| Intersect |         创建包含两个几何图形共有区域的形状          |
|    Xor    | 创建包含两个几何图形非共有区域的形状。相当于先合并，在移除共有部分  |
|  Exclude  | 创建的形状包含第一个几何图形的所有区域，但不包含第二个几何图形的区域 |

- 示例：
```XMAL

 <Path Fill="Red" Stroke="Blue" Margin="5">
     <Path.Data>
         <CombinedGeometry GeometryCombineMode="Xor">
             <CombinedGeometry.Geometry1>
                 <RectangleGeometry Rect="0,0 100,100"/>
             </CombinedGeometry.Geometry1>
             <CombinedGeometry.Geometry2>
                 <EllipseGeometry Center="85,50" RadiusX="65" RadiusY="35"/>
             </CombinedGeometry.Geometry2>
         </CombinedGeometry>
     </Path.Data>
 </Path>
```

CombinedGeometry类只能合并两个形状，这看起来可能是一个重大的局限，但并非如此。可构建包含许多不同几何图形的形状——只需要使用嵌套的CombinedGeometry对象即可。
- 示例：
```XAML

 <Path Fill="Yellow" Stroke="Blue" Margin="10">
     <Path.Data>
         <CombinedGeometry GeometryCombineMode="Union">
             <CombinedGeometry.Geometry1>
                 <CombinedGeometry GeometryCombineMode="Exclude">
                     <CombinedGeometry.Geometry1>
                         <EllipseGeometry Center="50,50" RadiusX="50" RadiusY="50"/>
                     </CombinedGeometry.Geometry1>
                     <CombinedGeometry.Geometry2>
                         <EllipseGeometry Center="50,50" RadiusX="40" RadiusY="40"/>
                     </CombinedGeometry.Geometry2>
                 </CombinedGeometry>
             </CombinedGeometry.Geometry1>

             <CombinedGeometry.Geometry2>
                 <RectangleGeometry Rect="44,5 10,90">
                     <RectangleGeometry.Transform>
                         <RotateTransform Angle="45" CenterX="50" CenterY="50"/>
                     </RectangleGeometry.Transform>
                 </RectangleGeometry>
             </CombinedGeometry.Geometry2>
         </CombinedGeometry>
     </Path.Data>
 </Path>
```
## 13.1.4使用PathGeometry绘制曲线和直线
PathGeometry是功能超级强大的内容，它能绘制其他所有几何图形能够绘制的内容，也能绘制其他所有几何图形不能绘制的内容。每个PathGeometry对象都是由一个或多个PathFigure对象构建的。每个PathFigure队形是一系列相互连接的直线和曲线，可闭合也可不闭合。
PathFigure类包含4个重要属性：

|     名称     | 说明                              |
| :--------: | :------------------------------ |
| StartPoint | 指示从何处开始绘制图形线条的Point对象           |
|  Segments  | 用于绘制图形的PathSegment对象的集合         |
|  IsClosed  | 如果为true，WPF添加直线来连接起点和终点         |
|  IsFilled  | 如果为true，就使用Path.Fill画刷填充图形内部的区域 |

PathFigure对象是由包含大量线段的不间断线条绘制的形状。然而，技巧有几种类型的线段，它们都继承自PathSegment类。

|             名称             | 说明                                                          |
| :------------------------: | :---------------------------------------------------------- |
|        LineSegment         | 在两点之间创建直线                                                   |
|         ArcSegment         | 在两点之间创建椭圆形弧线                                                |
|       BezierSegment        | 在两点之间创建贝塞尔曲线                                                |
|   QuadraticBezierSegment   | 创建形式更简单的贝塞尔曲线，只有一个控制点而不是两个控制点，并且计算速度更快                      |
|      PolyLineSegment       | 创建一系列直线。可使用多个LineSegment对象得到相同的效果，但使用单个PolyLineSegment对象更简明 |
| PolyQuadraticBezierSegment | 创建一些列更简单的二尺贝塞尔曲线                                            |
- 1.直线示例
```XAML

 <Path Stroke="Blue"  StrokeThickness="3">
     <Path.Data>
         <PathGeometry>
             <PathFigure IsClosed="True" StartPoint="10,100">
                 <LineSegment Point="100,100"/>
                 <LineSegment Point="100,50"/>
             </PathFigure>
         </PathGeometry>
     </Path.Data>
     <Path.Fill>
         <SolidColorBrush Color="Red"/>
     </Path.Fill>
 </Path>
```

- 2.弧线
```XAML
//size:虚拟椭圆大小的X轴,Y轴
//IsLargeArc：选择长短弧线，默认为flase选择短的一条弧线
//SweepDirection:翻转弧线
<Path Stroke="Blue" StrokeThickness="3" Margin="100">
    <Path.Data>
        <PathGeometry>
            <PathFigure IsFilled="True" StartPoint="10,100">
                <ArcSegment Point="250,150" Size="200,300" IsLargeArc="True" SweepDirection="Clockwise"/>
            </PathFigure>
        </PathGeometry>
    </Path.Data>
</Path>
```
- 3.贝塞尔曲线
  在起点，贝塞尔曲线和从第一个控制点到起点之间的直线相切。在终点，贝塞尔曲线和连接终点与最后一个点的直线相切。
   弯曲程度由到两个控制点的距离决定。如果一个控制点更远，该控制点会更强地”拉“贝塞尔曲线。
```XAML

   <Canvas>
       <Path Stroke="Blue" StrokeThickness="5" Canvas.Top="20">
           <Path.Data>
               <PathGeometry>
                   <PathFigure StartPoint="10,10">
                       <BezierSegment Point1="130,30" Point2="40,140" Point3="150,150"/>
                   </PathFigure>
               </PathGeometry>
           </Path.Data>
       </Path>
       <Path Stroke="Green" StrokeThickness="2" StrokeDashArray="5 2" Canvas.Top="20">
           <Path.Data>
               <GeometryGroup>
                   <LineGeometry StartPoint="10,10" EndPoint="130,30"/>
                   <LineGeometry StartPoint="40,140" EndPoint="150,150"/>
               </GeometryGroup>
           </Path.Data>
       </Path>
       <Path Fill="red" Stroke="Blue" StrokeThickness="8" Canvas.Top="20">
           <Path.Data>
               <GeometryGroup>
                   <EllipseGeometry Center="130,30"/>
                   <EllipseGeometry Center="40,140"/>
               </GeometryGroup>
           </Path.Data>
       </Path>
   </Canvas>
```

## 13.1.5微语言几何图形
WPF创作人员为定义几何图形增加了一种更简明的替换语法，通过改语法可用更少的标记表示详细的图形。被称为图形微语言又称路径微语言。
参数表格：

|                           名称                            | 说明                                                                                     |
| :-----------------------------------------------------: | :------------------------------------------------------------------------------------- |
|                         F value                         | 设置Geometry.FillRule属性。0表示EvenOdd,1表示NonZero。如果决定使用海命令，就必须将该命令存放在字符串的开头                 |
|                          M x,y                          | 为几何图形创建新的PathFigure对象，并设置其起点。该命令必须在其他命令之前使用，F命令除外。然而，也可在绘制序列期间使用该命令移动坐标系统的原点（M 代表Move) |
|                          L x,y                          | 创建一条到指定点的LineSegment几何图形                                                               |
|                           H x                           | 使用指定的X值创建一条水平的LineSegment几何图形，并保持Y值不变                                                  |
|                           V y                           | 使用指定的Y值创建一条垂直的LineSegment几何图形，并保持X值不变                                                  |
| A radius X,Radius Y degress isLargeArc, isClockwise x,y | 创建一条到指定点的ArgSegment线段。指定描述互相的椭圆半径、弧线旋转的度数，以及用于设置前面介绍的IsLargeArc和SweepDirection属性的布尔标志  |
|                    C x1,y1 x2,y2 x,y                    | 创建到指定点的贝塞尔曲线，使用（x1,y1）和（x2,y2)作为控制点                                                    |
|                      Q x1,y1 x2,y2                      | 创建到指定点的二次贝塞尔曲线，使用一个控制点（x1,y1)                                                          |
|                       S x2,y2 x,y                       | 通过将前一条贝塞尔曲线的第二个控制点作为新建贝塞尔曲线的第一个控制点，从而创建一条光滑的贝塞尔曲线                                      |
|                            Z                            | 结束当前PathFigure对象，并将IsClosed属性设置为Ture。                                                  |
```XAML
//实际上创建的是StreamGeometry对象，若要创建PathGeometry对象，应显示编写
 <Path Stroke="Blue" StrokeThickness="5" Data="M 10,100 L 100,100 L 100,50 Z"/>
```

==注意：如果希望命令的参数值相对于前一个点，而不是使用绝对坐标进行计算，可使用小写的命令==
## 13.1.6使用几何图形进行剪切
几何图形另一个用途是用于设置Clip属性，所有袁术都提供了该属性。可通过Clip属性约束元素的外边界以符合特定的集合图形。

```XAML
 <Window.Resources>
     <GeometryGroup x:Key="clipGeometry" FillRule="Nonzero">
         <EllipseGeometry RadiusX="75" RadiusY="50" Center="100,150"/>
         <EllipseGeometry RadiusX="100" RadiusY="25" Center="200,150"/>
         <EllipseGeometry RadiusX="75" RadiusY="130" Center="140,140"/>
     </GeometryGroup>
 </Window.Resources>
 
   <Button Clip="{ StaticResource clipGeometry}" Content="A button"/>
```

# 13.2图画
GeometryDrawing类不是WPF中唯一的图画类。实际上，Drawing类用于表示所有类型的2D图形，并且还有一组继承自该类。

| 类 | 说明 | 主要属性 |
|---|---|---|
| **GeometryDrawing** | 封装一个几何图形，并为其指定填充画刷和边框画笔，用于绘制矢量图形。 | Geometry, Brush, Pen |
| **ImageDrawing** | 封装一幅图像，并使用矩形区域定义图像在界面中的显示位置和大小。 | ImageSource, Rect |
| **VideoDrawing** | 结合媒体播放器播放视频，并通过矩形区域定义视频的显示范围。 | Player, Rect |
| **GlyphRunDrawing** | 封装底层文本对象（GlyphRun），用于以较底层方式绘制文本。 | GlyphRun, ForegroundBrush |
| **DrawingGroup** | 用于组合多个 Drawing 对象，可创建包含多种图形元素的复合图画，并能对整体应用效果。 | BitmapEffect, BitmapEffectInput, Children, ClipGeometry, GuidelineSet, Opacity, OpacityMask, Transform |

示例：
```XAML
    <Button>
        <Image>
            <Image.Source>
                <DrawingImage>
                    <DrawingImage.Drawing>
                        <GeometryDrawing>
                            <GeometryDrawing.Geometry>
                                <PathGeometry>
                                    <PathFigure IsClosed="True" StartPoint="10,100">
                                        <LineSegment Point="20,60"/>
                                        <LineSegment Point="80,150"/>
                                    </PathFigure>
                                </PathGeometry>
                            </GeometryDrawing.Geometry>
                            <GeometryDrawing.Pen>
                                <Pen Thickness="2"  Brush="Blue"/>
                            </GeometryDrawing.Pen>
                            <GeometryDrawing.Brush>
                                <SolidColorBrush Color="Red"/>
                            </GeometryDrawing.Brush>
                        </GeometryDrawing>
                        
                    </DrawingImage.Drawing>
                </DrawingImage>
            </Image.Source>
        </Image>
    </Button>
</StackPanel>
```
