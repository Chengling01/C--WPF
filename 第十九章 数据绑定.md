# 数据绑定
## 19.1 具有null值的绑定
绑定null值的结果是可预测的：目标元素不会显示任何内容，对于数字段，这一行为是非常有用的，因为他能区分缺少数值和零值的情况。在XAML中我们可以通过TargetBullValue属性来配置绑定值出现null的默认显示。
示例：
```XAML
Text="{Binding Path=Desctiption,TargetNullValue=[NO Description Provided]}"
```

## 19.3提高列表的性能
### 19.3.1WPF 虚拟化功能的作用

WPF 的虚拟化（Virtualization）指的是：**只创建当前可见区域所需的 UI 元素，而不为所有数据项生成对应的 UI 控件**。

作用：

- 减少内存占用
- 减少控件加载时间
- 增强大数据量场景下的滚动性能


### 19.3.2 WPF 虚拟化的核心实现机制

#### 19.3.2.1 VirtualizingStackPanel 控制 UI 虚拟化

虚拟化依赖 VirtualizingStackPanel，它负责：

- 确定可视区域
- 决定哪些 Item 创建 UI 容器
- 回收不可见项的 UI 容器（在 Recycling 模式中）

#### ItemContainerGenerator 动态生成和回收 UI 容器

ItemsControl 内部通过 ItemContainerGenerator 在滚动时动态创建或回收 ListBoxItem / ListViewItem 等容器，实现按需加载。

### 如何开启虚拟化

虚拟化通常默认开启，但在一些场景下需要手动配置。

#### 基础启用方式

```XMAL

<ListBox  VirtualizingStackPanel.IsVirtualizing="True"     VirtualizingStackPanel.VirtualizationMode="Recycling"     ScrollViewer.CanContentScroll="True"> </ListBox>

```
配置说明：
- IsVirtualizing：启用虚拟化
- VirtualizationMode="Recycling"：回收已存在的 UI 容器（性能更好）
- ScrollViewer.CanContentScroll="True"：允许内容虚拟化，而不是像素滚动

#### 使用 VirtualizingStackPanel 作为 ItemsPanel

若自定义 ItemsPanel，必须保持：

```XAML
<ListBox.ItemsPanel> 
    <ItemsPanelTemplate>   
          <VirtualizingStackPanel/>
     </ItemsPanelTemplate> 
</ListBox.ItemsPanel>`
```

如果换成 WrapPanel、Grid 等面板，将自动失效。


### #### 19.3.2.2 会导致虚拟化失效的情况

#### 常见导致虚拟化不生效的原因：
1. 使用 WrapPanel、Grid 等不支持虚拟化的面板作为 ItemsPanel
2. ScrollViewer.CanContentScroll 设置为 False（或被控件默认关闭）
3. 在控件模板或样式中修改了滚动逻辑，导致 ScrollViewer 发生变化
4. 项的高度/宽度不规则或无法确定
5. 使用 GroupStyle 而未开启分组虚拟化

#### 支持虚拟化的 WPF 控件列表

以下控件天然支持虚拟化，只要保持 VirtualizingStackPanel 和 CanContentScroll：
- ListBox:支持 UI 虚拟化，可与 ItemsPanel = VirtualizingStackPanel 配合工作。
- istView:支持虚拟化，即使使用 GridViewColumn 仍可正常虚拟化。
- TreeView:支持虚拟化，但通常需要开启模板中的虚拟化设置。

#### ComboBox（下拉部分）

下拉弹出列表支持虚拟化（使用 VirtualizingStackPanel）。
#### DataGrid

支持行虚拟化、列虚拟化、单元格虚拟化：

```XAML
<DataGrid      EnableRowVirtualization="True"     EnableColumnVirtualization="True"     VirtualizingPanel.IsVirtualizing="True"     VirtualizingPanel.VirtualizationMode="Recycling"/>
```

#### ItemsControl（只要 ItemsPanel 使用 VirtualizingStackPanel）

需要手动设置 ItemsPanel。

### 不支持虚拟化的布局面板

以下面板本身不支持虚拟化，因此作为 ItemsPanel 会禁用虚拟化：

- WrapPanel
- UniformGrid
- Canvas
- Grid（作为 ItemsPanel 不支持虚拟化）

### 总结

- WPF 虚拟化依赖 VirtualizingStackPanel 实现按需创建 UI 元素
- 开启虚拟化最重要的三个属性：  
    VirtualizingStackPanel.IsVirtualizing、VirtualizingStackPanel.VirtualizationMode、ScrollViewer.CanContentScroll
- 支持虚拟化的控件包括 ListBox、ListView、TreeView、ComboBox、DataGrid 等
- WrapPanel 等面板会导致虚拟化失效


### 19.3.2.3容器再循环
通常当滚动支持虚拟化列表时，控件不断地创建新的项容器对象已保存新的可见项。项容器再循环提高了滚动性能，降低了内存消耗，因为垃圾收集器不需要查找旧的项对象并释放它们。
示例：
```XAML
<ListBox VirtualizingStackPanel.VirtualizationMode="Recycling"/>
```

### 19.3.3缓存长度
可使用CacheLength和CacheLengthUnit这两个VirtualizingStackPanel属性进一步调整精确书来给你。CacheLength允许选择指定的附加项的数量：项数、页数或者像素，
示例：
```XAML
//在当前可见项之前和之后存储100项，之后存储有100项目
<ListBox VirtualizingStackPanel.CacheLength="100" VirtualizingStackPanel.CacheLengthUnit="Item"/>
```

```XMAL

//在当前可见项之前和之后存储100项，之后存储有500项目
<ListBox VirtualizingStackPanel.CacheLength="100,500" VirtualizingStackPanel.CacheLengthUnit="Item"/>
```

### 19.3.3滚动延迟
使用延迟滚动特性，当用户在滚动体条上拖动滚动滑块式不会更新列表显示，只有当用户释放了滚动滑块时才刷新。
示例：
```XAML

<ListBox ScrollViewer.IsDeferredScrollingEnabled="ture"/>
```

==配置滚动行为：Pixel(基于像素滚动)、Item(基于项滚动)==
示例：
```XAML

<ListBox VirtualizingStackPanel.ScrollUnit="Pixel"/>
```
## 19.4验证
WPF提供了能与前面讨论过的数据绑定系统紧密协作的验证功能。验证另外提供了以下两种方法用于捕获非法值：
- 可在数据对象中引发错误：为告知WPF发生了错误，只需要从属性设置过程中抛出异常。通常，WPF会忽略所有在设置属性时抛出的异常，但可以配置，从而显示更有帮助的可视化指示。另一种选择时在自定义的数据类中实现INotifyDataErrorInfo或者IDataErrorInfo接口，从而可得到指示时发生的错误的功能而不会抛出异常。
- 可在绑定级别上定义验证：这种方法可获得使用相同的验证的灵活性，而不必考虑使用的时那个输入控件。更好的是，因为是在不同的类中定义验证，所有可很容易地在存储类似数据类型的多个绑定中重用验证。

### 19.4.1在数据对象中进行验证
示例：

使用这种验证方式时，WP会不加通告地忽略当设置和获取属性时发生的数据绑定错误。对于这种情况，用户无法知道更新已经被拒绝。实际上，非法值任然保存在文本框中，只是没有被应用于绑定的数据对象。

对于每个失败的验证规则，WPF将采取以下三个步骤：
- 在绑定元素上（在此TextBox控件），将Validation.HasError附加属性设为True
- 创建包含错误细节的ValidationError对象（作为ValidationRule.Validate()方法的返回值），并将该对象添加到关联到的Validation.Errors集合中
- 如果Binding.NotifyOnValidationError属性被设置为true，WPF就在元素上引发Validation.Error附加事件（当配置这项信息时，验证失败就会调用Validation.Error事件处理程序）

#### 2.INotifyDataErrorInfo接口（通知错误接口）
`NotifyDataErrorInfo` 是 WPF 用来进行 **数据验证（Data Validation）** 的接口。
MVVM 模式中最推荐的验证方式。它允许 ViewModel：
- 主动告诉 UI 某个属性是否有错误
- 返回多个错误
- 异步验证（支持后台执行）
- 验证不是靠抛异常，而是靠“错误集合

与ExceptionValidationRule的比较：

| 功能        | ExceptionValidationRule | INotifyDataErrorInfo |
| --------- | ----------------------- | -------------------- |
| 后台赋值触发验证  | 不支持                     | 支持                   |
| 验证逻辑写在 VM | 规则写在 XAML               | 全部在 ViewModel        |
| 多个错误      | 只支持一个                   | 支持多个                 |
| 异步验证      | 不支持                     | 支持                   |
| MVVM 友好   | 否                       |  完全 MVVM             |

接口展示：
```Csharp

public interface INotifyDataErrorInfo
{
    bool HasErrors { get; }
    event EventHandler<DataErrorsChangedEventArgs> ErrorsChanged;

    IEnumerable GetErrors(string propertyName);
}

```

完整示例：
```Csharp

public class Person : INotifyPropertyChanged, INotifyDataErrorInfo
{
    private readonly Dictionary<string, List<string>> _errors 
        = new Dictionary<string, List<string>>();

    private int _age;
    public int Age
    {
        get => _age;
        set
        {
            _age = value;
            ValidateAge();
            OnPropertyChanged(nameof(Age));
        }
    }

    private void ValidateAge()
    {
        ClearErrors(nameof(Age));

        if (Age < 0)
            AddError(nameof(Age), "年龄不能为负数");

        if (Age > 120)
            AddError(nameof(Age), "年龄不能大于 120");
    }

    #region INotifyDataErrorInfo
    public bool HasErrors => _errors.Any();

    public event EventHandler<DataErrorsChangedEventArgs> ErrorsChanged;

    public IEnumerable GetErrors(string propertyName)
    {
        if (string.IsNullOrEmpty(propertyName))
            return null;

        if (_errors.ContainsKey(propertyName))
            return _errors[propertyName];

        return null;
    }
    #endregion

    #region Error Helpers
    private void AddError(string propertyName, string error)
    {
        if (!_errors.ContainsKey(propertyName))
            _errors[propertyName] = new List<string>();

        _errors[propertyName].Add(error);

        ErrorsChanged?.Invoke(this, new DataErrorsChangedEventArgs(propertyName));
    }

    private void ClearErrors(string propertyName)
    {
        if (_errors.ContainsKey(propertyName))
        {
            _errors.Remove(propertyName);
            ErrorsChanged?.Invoke(this, new DataErrorsChangedEventArgs(propertyName));
        }
    }
    #endregion

    #region INotifyPropertyChanged
    public event PropertyChangedEventHandler PropertyChanged;

    protected void OnPropertyChanged(string name)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
    #endregion
}

```

### 19.4.2自定义验证规则
自定义验证规则的方法和应用自定义转换器的方法类似。该方法继承自validationRule的类，并为了执行而重写Validate（）方法。
示例：
```Csharp

public class PositivePriceRule:ValidationRule
{
    private decimal min = 0;
    private decimal max = Decimal.MaxValue;
    public decimal Min { get { return min; } set { min = value; } }
    public decimal Max { get { return max; } set { max = value; } }

    public override ValidationResult Validate(object value, CultureInfo cultureInfo)
    {
        decimal price = 0;
        try
        {
            if(((string)value).Length>0)
            {
                price=Decimal.Parse((string)value,NumberStyles.Any,cultureInfo);
            }
        }catch
        {
            return new ValidationResult(false, "Tllegal characters");
        }

        if (price < min || price > max)
        {
            return new ValidationResult(false, "Not in the range" + Min + "to" + Max + ".");
        }
        else
        {
            return new ValidationResult(true, null);
        }
    }
}
```

### 19.4.6验证多个值
在 WPF 中，**使用绑定组（BindingGroup）来自定义验证规则**是一种非常强大且适用于**跨多个绑定属性的联合验证**的技术。它特别适合表单级验证（例如：密码与确认密码一致、开始时间早于结束时间等），而无需依赖 ViewModel 中复杂的 `INotifyDataErrorInfo` 逻辑（当然两者也可结合）。

#### 19.4.6.1什么是 BindingGroup？

`BindingGroup` 是 WPF 中的一个类，用于将多个绑定组合在一起，并对整个组执行**提交前验证（Commit Validation）** 或 **值更改时验证（Value Validation）**。

- 它允许你编写一个 `ValidationRule`，该规则可以访问组内所有绑定的值。
- 验证发生在用户尝试“提交”数据时（如点击保存按钮），或在值改变时（取决于设置）。
- 常用于 `Grid`、`StackPanel` 等容器上，包裹多个输入控件。

---

#### 19.4.6.2核心步骤

1. 在 XAML 中为容器（如 `StackPanel`）设置 `BindingGroup`。
2. 为每个子控件的绑定指定 `BindingGroupName`。
3. 创建自定义 `ValidationRule`，重写 `Validate` 方法，访问 `BindingGroup` 中的所有项。
4. 在需要时（如点击按钮）调用 `BindingGroup.CommitEdit()` 触发验证。

#### 19.4.6.3完整示例：验证“开始时间 < 结束时间”

##### 场景

用户输入开始时间和结束时间，点击“保存”时验证逻辑。

### 3. 自定义验证规则（TimeRangeValidationRule.cs）

```
using System;
using System.Globalization;
using System.Windows.Controls;
using System.Windows.Data;

namespace WpfBindingGroupDemo
{
    public class TimeRangeValidationRule : ValidationRule
    {
        public override ValidationResult Validate(object value, CultureInfo cultureInfo)
        {
            // value 是 BindingGroup 对象
            if (value is not BindingGroup bindingGroup)
                return new ValidationResult(false, "内部错误：未获取到 BindingGroup");

            // 从 BindingGroup 获取各个绑定项的值
            object startValue = null, endValue = null;

            foreach (var item in bindingGroup.Items)
            {
                // item 是绑定源对象（这里是 MainViewModel 实例）
                if (item is MainViewModel model)
                {
                    startValue = model.StartTime;
                    endValue = model.EndTime;
                    break;
                }
            }

            // 如果任一为空，跳过验证（或根据需求处理）
            if (!startValue?.Equals(null) == true || !endValue?.Equals(null) == true)
                return ValidationResult.ValidResult;

            if ((DateTime)startValue > (DateTime)endValue)
            {
                return new ValidationResult(false, "开始时间不能晚于结束时间！");
            }

            return ValidationResult.ValidResult;
        }
    }
}
```

> 提示：也可以通过 `bindingGroup.Bindings` 访问每个 `BindingExpression`，但直接访问源对象（ViewModel）更简单。

### 4. 后台代码（MainWindow.xaml.cs）


```
using System.Windows;

namespace WpfBindingGroupDemo
{
    public partial class MainWindow : Window
    {
       private readonly MainViewModel _viewModel = new();

        public MainWindow()
        {
            InitializeComponent();
            DataContext = _viewModel;
        }

        private void SaveButton_Click(object sender, RoutedEventArgs e)
        {
            var bg = this.FindName("TimeGroup") as BindingGroup; //  错误！BindingGroup 没有 Name 属性

           // 正确方式：从 StackPanel 获取 BindingGroup
           var panel = (StackPanel)this.Content;
            var bindingGroup = panel.BindingGroup;

            if (bindingGroup.CommitEdit())
            {
                MessageBox.Show("数据有效，已保存！");
            }
           else
            {
                // 验证失败，WPF 会自动显示错误（需模板支持）
                // 可手动遍历错误：
                foreach (var error in bindingGroup.ValidationErrors)
                {
                    MessageBox.Show($"验证失败: {error.ErrorContent}");
                    break;
                }
            }
        }
    }
}
```

> 注意：`BindingGroup` **没有 `Name` 属性**，不能用 `FindName`。应直接从容器（如 `StackPanel`）获取 `.BindingGroup`。

#### 19.4.6.4优点与适用场景

|优点|说明|
|---|---|
|**集中验证**|所有相关字段在一个规则中处理|
|**UI 与逻辑解耦**|验证规则独立，可复用|
|**天然支持提交时验证**|避免频繁触发验证|
|**自动错误提示**|若 TextBox 使用默认样式，会显示红色边框|

|适用场景|
|---|
|表单提交前验证（如注册、订单）|
|多字段互斥/依赖逻辑（如日期范围、密码确认）|
|不想在 ViewModel 中写复杂验证逻辑时|

#### 19.4.6.5注意事项

1. **BindingGroup 默认不自动验证**，需调用 `CommitEdit()` 或设置 `ValidatesOnTargetUpdated="True"`（不推荐，性能差）。
2. 验证规则中的 `value` 参数是 `BindingGroup` 对象。
3. 若要实时验证，可在属性变更时手动调用 `UpdateSources()` + `Validate()`.
4. 错误信息默认不会显示在界面上，除非控件模板支持（如 TextBox 默认会显示红色边框和 tooltip）。
