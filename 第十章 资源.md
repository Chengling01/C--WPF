# 资源
## 10.1 资源
**`WPF`允许在代码中以及在标记中的各个位置定义资源（和特定的控件、窗口一起定义，或在整个应用程序中定义）。**
资源具有许多重要的优点，如下所述：
- **高效。** 可以通过资源定义对象，并标记中的多个地方使用。在这种模式下，会精简代码，使其更加高效。
- **可维护性。** 可通过资源使用低级的格式化细节（如字号），并将它们移到便于对其进行修改的中央位置。在XAML中创建资源相当于代码中创建常量。
- **适应性。** 一旦特定信息与应用程序的其他部分分离开来，并放置到资源的部分中，就可以动态地修改这些信息。
### 10.1.1资源集合
每个元素都有Resources属性，该属性存储了一个资源字典集合（ResourceDictionary类的实例）。==但通常在窗口定义资源，这是因为每个元素都可以访问个自资源集合中的资源吗，也可以访问所有父元素的资源集合中的资源。==
简单示例：
```XAML
//在窗口下定义一个图片画刷资源
<Window.Resources>
  <ImageBrush x:Key="TileBrush" TileMode=“Tile" ViewportUnits="Absolute" Viewport="0 0 32 32" ImageSource="happyface.jpg" Opacity="0.3"/>
</Window.Resource>
```

```XAML
<Button Backgroud="{StaticResource TileBrush}" Magin="5" Padding="5" FontWeight="Bold" FontSize="14" Content="A Tiled Button"/>
```
### 10.1.2静态资源和动态资源
静态资源的设置在首次创建窗口时一次性完成，并且自从资源集合中获取一次对象，这以为着资源发生改变时，引用资源的控件无法做出改变。而动态资源在每次需要对象时都会重新从资源集合中查找对象。
作为一般规则，只有在下列情形下才需要使用动态属性：
- 资源具有依赖于系统设置的属性（如当前Windows操作系统的颜色或字体）。
- 准备通过编程方式替换对象。
- 
==以下提供通过代码的方式来获取资源集合中的资源：==
```csharp
//TileBrush:资源名
ImageBrush brush=(ImageBrush)this.Resources["TileBrush"];
brush.Viewport=new Rect(0,0,5,5);
```
==通过代码来访问资源，可使用FrameworkElement.FindResource(),也可以使用FrameworkElement.FindResource()来防止找不到资源时报错，示例如下：==
```csharp
private void camChange_Click(object sender,RoutedEventArgs)
{
  Button cmd=(Button)sender;
  ImageBrush brush=(ImageBrush)sender.FindResource("TileBrush");
}
```
### 10.1.3应用程序资源
窗口不是查找资源的最后一站，如果在控件或其容器中都找不到指定资源，WPF会继续检查未应用程序定义的资源集合，但为了平衡系统的重用性和复杂性，在对象需要广泛重用时，可使用应用程序资源。而定义应用程序资源实在App.xaml文件内定义。
```XAML
<Application.Resources>
  <ImageBrush x:Key="TileBrush" TileMode="Tile" ViewportUnits="Absoulute" Viewport="0 0 32 32"/>
</Application.Resources>
```
### 10.1.4系统资源
==系统资源常常使用静态属性公开它们的细节，在引用时需要通过引用静态资源的方式获取，但该种方法在应用程序运行时，改变Windows颜色时，控件不会更新本身。因而提出以下解决方法：==
```XAML
//通过使用ResourceKey对象引用的补充属性集，从系统资源集合中提取资源，可解决以上问题。
<Label Foreground="{DynamicResource {x:Static SystemColors.WindowTextBrushKey}}/>
```

## 10.2资源字典
如果希望在多个项目之间共享资源，可创建资源字典。资源字典只是XAML文档，除了存储希望使用的资源外，不可做其他任何事情。
### 10.2.1创建资源字典
在解决方法项目下右键鼠标，选中创建资源字典文件。在资源字典内书写想要定义的资源。
```XAML
<ResourceDictionary>
......
<IamgeBrush X:Key="TileBrush" TileMode="Tile" ViewportUnits="Absolute" Viewport="0 0 32 32" ImageSource="happyface.jpg" opacity="0.3"/>
/ResourceDictionary>
```
### 10.2.2使用资源字典
为了使用资源字典，需要将其合并到应用程序某些位置的资源集合中。例如：
```XAML
<Application x:Class="Resources.App"
......
  <Application.Resource>
    <ResourceDictionary>
      <ResourceDictionary.MergedDictionaries>
      //应用程序引用的程序资源
        <ResourceDictionary Source="AppBrushes.xaml"/>
        <ResourceDictionary Source="WizardBrushes.xaml"/>
      </ResourceDictionary.MergedDictionaries>
      //可在前项节点前后写入本地资源
    </ResoutceDictionary>
  </Application.Resource>
</Application>
```
