# 样式和行为
**`样式（Style）`** 是组织和重用格式化选项的重要工具。不是使用重复标记填充XAML，以便设置外边距、内边距、颜色以及字体等细节，而是创建一系列封装所有这些细节的样式，然后在需要之处通过属性来应用样式。
**`行为（Behavior）`** 是一款重用用户界面代码的更有挑战性的工具。其基本思想是：使用行为封装一些通用的UI功能（例如：使用可被拖动的代码）。
## 11.1样式基础
样式是可应用于元素的集合的属性值集合。同样样式具有属性值继承的特性，当在窗口级别设置这些属性时，窗口中的所有元素都会使用相同的属性值，除非明确地覆盖它们。在定义Key值时，根据约定以Style结尾。

样式的基本属性：

|        属性        |                                                 说明                                                  |
| :--------------: | :-------------------------------------------------------------------------------------------------: |
|   Setters（设置器）   |                            设置属性值以及自动关联事件处理程序的Setter对象或者EventSetter对象的集合                             |
|  Triggers（触发器）   |                  继承自TriuggerBase类并能自动改变样式设置的对象集合。例如，当另一个集合属性改变时，或者当某个事件发生时，可以修改样式                   |
|  Resources(资源)   | 希望用于样式的资源集合。例如，可能需要使用一个对象设置多个属性。这时，更高效的做法是作为资源创建对象，然后再Setter对象中使用资源（而不是使用嵌套的标记作为每个Setter对象的一部分创建对象） |
|   BasedOn(继承于)   |                                 通过该属性可以创建继承自其他样式设置的更具体的样式，并且可以实现重写                                  |
| TargetType(目标类型) |                   该属性表示应用样式的元素的类型。通过该属性可创建只影响特定类型元素的设置器，还可以创建能过为恰当的元素类型自动器作用的设置器                    |
### 11.1.1创建样式对象
如果希望创建目标更加精细的样式，可使用容器的Resources集合定义样式。如果希望在应用程序中重用样式，可使用应用程序的Resources集合定义样式。
示例：
```XAML

<Button Padding="5" Margin="5">
  <Button.Style>
    <Style>
      <Setter Property="Control.FontFamily" Value="Time New Roman"/>
      <Setter Property="Control.FontSize" Value="18"/>
      <Setter Property="Control.FontWeight" Value="Bold"/>
    </Style>
  </Button.Style>
</Button>
```
### 11.1.2设置属性
在设置样式时，需要注意唯一的限制时设置器只能该白你依赖项属性——不能修改其他属性。同时也不能使用简单字符串创建ImageBrush对象。对于这种情况将采用嵌套的元素代替特性。
示例：
```XAML

<Style x:Kwy="HappyTiledElementStyle>
  <Setter Property="Control.BackGround">
    <Setter.Value>
      <ImageBrush TileMode="Tile" ViewPortUnits="Absolute" Viewport="0 0 32 32" ImageSource="happyface.jpg" Opacity="0.3"/>
    </Setter.Value>
  </Setter>
</Style> 
```
当需要为特定的控件来设置样式时，可使用TargetType属性，示例：
```XAML

//TargetType:目标类型
<Style x:Kwy="HappyTiledElementStyle TargetType="Button">
  <Setter Property="BackGround">
    <Setter.Value>
      <ImageBrush TileMode="Tile" ViewPortUnits="Absolute" Viewport="0 0 32 32" ImageSource="happyface.jpg" Opacity="0.3"/>
    </Setter.Value>
  </Setter>
</Style> 
```

### 11.1.3关联事件处理程序
属性设置器是所有样式中最常见的要素，但也可以创建为事件关联特定事件处理程序的EventSetter对象的集合。
示例：
```XAML

<Style x:Key="MouseOVerHighlightStyle">
  <EventSetter Event="TextBlock.MouseEnter" Handler="element_MouseEnter"/>
  <EventSetter Event="TextBlock.MouseLeave" Handler="element_MouseLeave"/>
  <Setter Property="TextBlock.Panding" Value="5"/>
</Style>
```
事件处理代码：
```csharp

private void element_MouseEnter(object sender,MouseEventArgs e)
{
  ((TextBlock)sender).Background=new SolidColorBrush(Colors.LightGoldenrodYellow);
}

private void element_MouseLeave(object sender,MouseEventArgs e)
{
  ((TextBlock)sender).Background=null;
}
```
### 11.1.4多层样式
由于属性值继承和样式继承的特性，在希望为一组控件使用相同的字体，又不想为每一控件应用相同样式时，可以将它们放置在面板中，并设置面板的样式即可实现该诉求。同时可通过BasedOn属性继承样式资源。
示例：
```XAML

<Window.Resources>
  <Style x:Key="BigFontButtonStyle">
    <Setter Property="Control.FontFamilt" Value="Times New Roman"/>
    <Setter Property="Control.FontSize" Value="17"/>
  </Style>
  
  <Style x:Key="EmphasizedBigFontButtonStyle" BasedOn="{StaticResource BigFontButtonStyle}">
    <Setter Property="Control.Foreground" Value="White"/>
    <Setter Property="Control.Background" Value="DarkBlue"/>
  </Style>
</Window.Resources>
```
==注意：同一属性值的子样式会覆盖父类样式==

### 11.1.5通过类型自动应用样式
WPF中可以通过类型自动应用样式，从而忽略键名，如下所示：
```XAML 
//当使用这种样式定义方式时，会自动将该样式范围内的所有Button控件采用该样式
<Window.Resources>
  <Style TargetType="Button">
    <Setter Property="FontFamily" Value="Times New Roman"/>
    <Setter Property="FontSize" Value="18"/>
  </Style>
</Window.Reources>
```
若需要删除引用这种样式时，给控件的Style赋予空即可”Style="{x:Null}""。

## 11.2触发器
WPF中可以以声明的方式扩展代码功能。当使用样式，资源或者数据绑定时，将发现即使不使用代码，也能完成不少工作。而触发器时另一个实现这种功能的例子。使用触发器，可自动完成简单样式的改变，而着通常使用样板事件处理逻辑。
触发器通过Style.Triggers集合链接到样式。每个样式都可以有任意多个触发器，而且每个触发器都是System.Windows.TridderBase的派生类的实例。

|        名称        | 说明                                        |
| :--------------: | ----------------------------------------- |
|     Trigger      | 这是一种最简单的触发器。可以监测依赖项属性的变化，然后使用设置器改变样式      |
|   MultiTrigger   | 与Trigger类似，但这种触发器联合多个条件。只有满足了所有条件，才会启动触发器 |
|   DataTrigger    | 这种触发器使用数据绑定。与Trigger类似，只不过监视的是任意绑定数据的变化   |
| MultiDataTrigger | 联合多个数据触发器                                 |
|   EventTrigger   | 这是最复杂的触发器。当事件发生时，这种触发器应用动画                |
### 11.2.1简单触发器
可为任何依赖项属性关联简单触发器，每个简单触发器都指定了正在监视的属性，以及正在等待的属性值。当该属性值出现时，将应用储存在Trigger.Setters集合中的设置器。
示例：
```XAML

<Style x:Key="BigFontButton">
  <Style.Setters>
    <Setter Property="Control.FontFamily" Value="Time New Roman"/>
    <Setter Property="Control.FontSize" Value="18"/>
  </Style.Setters>
  <Style.Triggers>
    <Trigger Property="Control.IsFocused" Value="True">
      <Setter Peoperty="Control.Foreground" Value="DarkRed"/>
    </Trigger>
  </Style.Triggers>
</Style>
```
==注意：当多个触发器修改同一属性，那么最后的触发器将有效。==

当希望创建只有满足多个条件时才激活的触发器，可使用MultiTigger。这种触发器提供了一个Conditions集合，可通过定义一系列属性和值的组合。
示例：
```XAML

<Style x:Key="BigFontButton">
  <Style.Setters>
    ......
  </Style.Setters>
  <Style.Triggers>
    <MultiTrigger>
      <MultiTrigger.Conditions>
        <Condition Property="Control.IsFocused" Value="True"/>
        <Condition Property="Control.IsMouseOver" Value="True"/>
      </MultiTrigger.Conditins>
      <MultiTrigger.Setters>
        <Setter Property="Control.Foreground" Value="DarkRed"/>
      </MultiTrigger.Setters>
    </MultiTrigger>
  </Style.Triggers>
</Style>
```
### 11.2.2事件触发器
普通触发器等待属性发生变化，而事件触发器等待特定的事件被引发。
示例：
```XAML

<Style x:Key="BigFontButtonStyle">
  <Style.Setters>
    ......
  </Style.Setters>
  <Style.Triggers>
    <EventTrigger RoutedEvent="Mouse.MouseEnter">
      <EventTrigger.Actions>
        <BeginStoryboard>
          <Storyboard>
            <DoubleAnimation Duration="0:0:0.2" Storyboard.TargetProperty="FontSize" To="22"/>
          </Storyboard>
        </BeginStoryboard>
      </EventTrigger.Actions>
    </EventTrigger>
</Style>
```
在XAML中，必须在故事板中定义每个动画，故事板为动画提供时间线。用户可以在故事板内部定义希望使用的一个或多个动画对象。每个动画对象执行本质上时相同的任务：在一定时间内修改依赖项属性。与属性触发器不同，如果希望元素返回到原始状态，需要反转事件触发器（这是因为默认的动画行为时一旦动画完成就继续处于激活状态，从而保持最后的属性值）。

### 11.3.1如何使用行为
对于WPF框架，使用行为前需要NuGet一下**Microsoft.Xaml.Behaviors.Wpf**。从而来实现对行为的使用和自定义。
示例：
```XAML

//引入资源：xmlns:i="http://schemas.microsoft.com/xaml/behaviors"
//该示例列举的是，让Border控件具有鼠标拖动的行为。
<Window x:Class="行为学习.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
        xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
        xmlns:local="clr-namespace:行为学习"
        xmlns:i="http://schemas.microsoft.com/xaml/behaviors"
        mc:Ignorable="d"
        Title="MainWindow" Height="450" Width="800">
    <Grid>
        <Border Width="100" Height="100" Background="Red">
            <i:Interaction.Behaviors>
                <i:MouseDragElementBehavior/>
                <local:MyBehavior/>
            </i:Interaction.Behaviors>
        </Border>
    </Grid>>
        
```
### 11.3.2自定义行为
在自定义行为时，可以掺杂属性和依赖属性的使用，从而提高自定义行为的高阶方法。
示例1：

```XAML

    <Grid>
        <Border Width="100" Height="100" Background="Red">
            <i:Interaction.Behaviors>
                <i:MouseDragElementBehavior/>
                <local:MyBehavior/>
            </i:Interaction.Behaviors>
        </Border>
    </Grid>
```

```csharp

    //自定义行为时：需要继承Behavior类，并指定泛型
    class MyBehavior : Behavior<Border>
    {
        //重写OnAttached方法：为当前行为的关联对象附加行为
        //调用时机：当行为（MyBehavior 实例）被附加到目标对象（这里是 Border 控件）时触发。
        //具体来说，当你在 XAML 中通过<i:Interaction.Behaviors> 为某个 Border 控件添加 MyBehavior，
        //或者在代码中通过 Behaviors.Add(new MyBehavior()) 关联时，OnAttached 会被自动调用。
        protected override void OnAttached()
        {
            //AssociatedObject:行为关联对象
            AssociatedObject.MouseEnter += (object_, MouseEventArgs_) => AssociatedObject.Background = Brushes.Orange;
            AssociatedObject.MouseLeave+= (object_, MouseEventArgs_) => AssociatedObject.Background = Brushes.Blue;
        }

        //重写OnDetaching方法：将当前行为的关联对象的附加行为解除
        //调用时机：当行为（MyBehavior 实例）从目标对象（Border 控件）上移除时触发。
        //例如，通过 Behaviors.Remove(behaviorInstance) 手动移除行为，或者当关联的 Border 控件被销毁时，OnDetaching 会被自动调用。
        protected override void OnDetaching()
        {
            //AssociatedObject:行为关联对象
            AssociatedObject.MouseEnter -= (object_, MouseEventArgs_) => AssociatedObject.Background = Brushes.Orange;
            AssociatedObject.MouseLeave -= (object_, MouseEventArgs_) => AssociatedObject.Background = Brushes.Blue;
        }
    }
```

示例2：
```XAML

    <StackPanel>
        <TextBox x:Name="tbox">
            <i:Interaction.Behaviors>
                <local:MouseWheelBehavior MaxValue="100" MinValue="0" Scale="5"/>
            </i:Interaction.Behaviors>
        </TextBox>
        <Button HorizontalAlignment="Left" Content="Clear">
            <i:Interaction.Behaviors>
                <local:ClearTextBehavior Target="{Binding ElementName=tbox}"/>
            </i:Interaction.Behaviors>
        </Button>
     </StackPanel>
```

```csharp

    class ClearTextBehavior:Behavior<Button>
    {
        public TextBox Target
        {
            get { return (TextBox)GetValue(TargetProperty); }
            set { SetValue(TargetProperty, value); }
        }

        // Using a DependencyProperty as the backing store for Target.  This enables animation, styling, binding, etc...
        public static readonly DependencyProperty TargetProperty =
            DependencyProperty.Register("Target", typeof(TextBox), typeof(ClearTextBehavior), new PropertyMetadata(null));

        protected override void OnAttached()
        {
            AssociatedObject.Click += ButtonClick;
        }

        private void ButtonClick(object sender, RoutedEventArgs e)
        {
            Target?.Clear();
        }
    }

```
