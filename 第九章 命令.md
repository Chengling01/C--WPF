# 命令
## 9.1WPF命令模型
### 9.1.0命令的基本概述

**WPF命令模型有许多可变的部分组成。总而言之，它们都具有如下4个重要元素：**
- **命令：** 命令表示应用程序任务，并且跟踪任务是否能够被执行。然而，命令并不包含执行应用程序任务的代码。
- **命令绑定：** 每个命令绑定针对用户界面的具体区域，将命令连接到相关的应用程序逻辑。这种分解的设计是非常重要的，因为单个命令可用于应用程序中的多个地方，并且在多个地方具有着不同的意义。为了处理这一问题，需要将同一命令与不同的命令绑定。
- **命令源：** 命令源触发命令。例如，**MenuItem**和**Button**都是命令源。单击它们都会执行绑定命令。
- **命令目标：** 命令目标是在其中执行命令的元素。例如，**Paste**命令可在**TextBox**控件中插入文本，而**OpenFile**命令可在**DocumentViewer**中打开文档。
### 9.1.1ICommand接口

**WPF命令模型的核心是`System.Windows.Input.ICommand接口`，该接口定义了命令的工作原理。该接口包含两个方法和一个事件：
```Csharp

Public interface ICommand
{
//命令执行方法
  void Execute(object parameter);
//命令状态检测方法
  bool CanExecute(object parameter);
//命令状态改变事件：当命令可用时，命令源可自动启用自身；当命令不可用时，禁用自身。
  event EventHandler CanExecuteChanged;
}
```
### 9.1.2RoutedCommand类
**当创建自己的命令时，不会直接实现`ICommand`接口；而是使用`System.Windows.Input.RoutedCommand`类，该类自动实现了`ICommand`接口。所以WPF命令都是`RoutedCommand`类及其派生类的实例。同时`RoutedCommand`类为事件冒泡程序和隧道添加了一些额外的基础，实现命令可在WPF元素层次结构中冒泡，以便获得正确的事件处理程序。**
以下时RoutedCommand的简要使用实例
#### **步骤 1：定义 RoutedCommand**
通常在静态类中定义全局命令，方便多个元素共享。

```csharp
using System.Windows.Input;

public static class CustomCommands
{
    // 定义路由命令（命令名称、所属类型、输入手势可选）
    public static readonly RoutedCommand Save = new RoutedCommand(
        "Save", 
        typeof(CustomCommands),
        new InputGestureCollection { new KeyGesture(Key.S, ModifierKeys.Control) } // 绑定Ctrl+S快捷键
    );
}
```

#### **步骤 2：在 XAML 中关联命令源**

将命令绑定到按钮、菜单项等元素（命令源），并指定命令目标（可选）。

```xaml
<Window x:Class="WpfApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:local="clr-namespace:WpfApp"
        Title="RoutedCommand示例" Height="300" Width="400">
    
    <!-- 命令源：按钮触发Save命令 -->
    <Button Content="保存" 
            Command="{x:Static local:CustomCommands.Save}" 
            CommandTarget="{Binding ElementName=txtEditor}" /> <!-- 命令目标为文本框 -->
    
    <!-- 命令目标：文本框（命令将作用于它） -->
    <TextBox x:Name="txtEditor" Margin="10" />
</Window>
```

#### **步骤 3：添加命令绑定（CommandBinding）**

在命令目标的上级元素（如`Window`、`Panel`）中定义`CommandBinding`，关联命令的执行逻辑和状态检查。

```xaml
<Window ...>
    <Window.CommandBindings>
        <!-- 绑定Save命令的执行逻辑 -->
        <CommandBinding 
            Command="{x:Static local:CustomCommands.Save}"
            Executed="Save_Executed"       
            CanExecute="Save_CanExecute"  
        />
    </Window.CommandBindings>

    <!-- 界面元素 -->
    <StackPanel>
        <Button Content="保存" Command="{x:Static local:CustomCommands.Save}" />
        <TextBox x:Name="txtEditor" Margin="10" />
    </StackPanel>
</Window>
```

#### **步骤 4：实现命令逻辑（后台代码）**

在`Executed`事件中编写实际业务逻辑，在`CanExecute`中判断命令是否可用（如文本不为空时允许保存）。

```csharp
public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();
    }

    // 命令执行逻辑
    private void Save_Executed(object sender, ExecutedRoutedEventArgs e)
    {
        string content = txtEditor.Text;
        // 执行保存操作（如写入文件）
        MessageBox.Show($"已保存内容：{content}");
        e.Handled = true; // 标记事件已处理，停止路由
    }

    // 命令可用性检查
    private void Save_CanExecute(object sender, CanExecuteRoutedEventArgs e)
    {
        // 文本不为空时允许执行命令
        e.CanExecute = !string.IsNullOrEmpty(txtEditor.Text);
        e.Handled = true;
    }
}
```
### 9.1.3RoutedUICommand类

**`RoutedUICommand`类用于具有文本的命令，这些文本显示在用户界面中的某些地方（例如：菜单项文本、工具栏按钮的工具提示）。`RoutedUICommand`类只是在`RoutedCommand`类的基础上添加了`Text`属性，该属性是为了命令显示文本。其优点在于可在某个位置执行本地化，但如果命令文本永远不会在用户界面的任何地方显示，那么它与`RoutedCommand`类等效的。**
以下时RoutedUICommand的简要使用实例
### **步骤 1：定义 `RoutedUICommand` 命令**

在静态类中定义带显示文本和快捷键的命令：

```csharp
// EditorCommands.cs
using System.Windows.Input;

namespace RoutedUICommandDemo
{
    public static class EditorCommands
    {
        // 新建命令：文本"新建"，快捷键Ctrl+N
        public static readonly RoutedUICommand New = new RoutedUICommand(
            "新建",       // Text属性（UI显示文本）
            "New",        // 命令名称
            typeof(EditorCommands),
            new InputGestureCollection { new KeyGesture(Key.N, ModifierKeys.Control) }
        );

        // 打开命令：文本"打开"，快捷键Ctrl+O
        public static readonly RoutedUICommand Open = new RoutedUICommand(
            "打开", 
            "Open", 
            typeof(EditorCommands),
            new InputGestureCollection { new KeyGesture(Key.O, ModifierKeys.Control) }
        );

        // 保存命令：文本"保存"，快捷键Ctrl+S
        public static readonly RoutedUICommand Save = new RoutedUICommand(
            "保存", 
            "Save", 
            typeof(EditorCommands),
            new InputGestureCollection { new KeyGesture(Key.S, ModifierKeys.Control) }
        );
    }
}
```

### **步骤 2：设计 XAML 界面**

使用菜单和按钮绑定命令，通过 `Command.Text` 自动显示命令文本：

```xaml
<!-- MainWindow.xaml -->
<Window x:Class="RoutedUICommandDemo.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:local="clr-namespace:RoutedUICommandDemo"
        Title="RoutedUICommand示例" Height="400" Width="600">

    <!-- 命令绑定：关联命令与执行逻辑 -->
    <Window.CommandBindings>
        <CommandBinding 
            Command="{x:Static local:EditorCommands.New}"
            Executed="New_Executed"
            CanExecute="AlwaysCanExecute" <!-- 始终可执行 -->
        />
        <CommandBinding 
            Command="{x:Static local:EditorCommands.Open}"
            Executed="Open_Executed"
            CanExecute="AlwaysCanExecute"
        />
        <CommandBinding 
            Command="{x:Static local:EditorCommands.Save}"
            Executed="Save_Executed"
            CanExecute="Save_CanExecute" <!-- 仅文本有内容时可执行 -->
        />
    </Window.CommandBindings>

    <DockPanel>
        <!-- 菜单：绑定命令，自动显示Command.Text -->
        <Menu DockPanel.Dock="Top">
            <MenuItem Header="文件">
                <MenuItem Command="{x:Static local:EditorCommands.New}" />
                <MenuItem Command="{x:Static local:EditorCommands.Open}" />
                <Separator />
                <MenuItem Command="{x:Static local:EditorCommands.Save}" />
            </MenuItem>
        </Menu>

        <!-- 工具栏：按钮文本绑定Command.Text -->
        <ToolBar DockPanel.Dock="Top">
            <Button Command="{x:Static local:EditorCommands.New}" 
                    Content="{Binding Command.Text, RelativeSource={RelativeSource Self}}" />
            <Button Command="{x:Static local:EditorCommands.Open}" 
                    Content="{Binding Command.Text, RelativeSource={RelativeSource Self}}" />
            <Button Command="{x:Static local:EditorCommands.Save}" 
                    Content="{Binding Command.Text, RelativeSource={RelativeSource Self}}" />
        </ToolBar>

        <!-- 文本编辑区域 -->
        <TextBox x:Name="txtEditor" AcceptsReturn="True" Margin="5" />

        <!-- 状态提示 -->
        <StatusBar DockPanel.Dock="Bottom" Height="25">
            <TextBlock x:Name="txtStatus" />
        </StatusBar>
    </DockPanel>
</Window>
```

### **步骤 3：实现命令逻辑**

在后台代码中编写命令的执行逻辑和可用性检查：

```csharp
// MainWindow.xaml.cs
using System.Windows;
using System.Windows.Input;

namespace RoutedUICommandDemo
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
        }

        // 新建命令执行：清空文本
        private void New_Executed(object sender, ExecutedRoutedEventArgs e)
        {
            txtEditor.Clear();
            txtStatus.Text = "已新建空白文档";
            e.Handled = true;
        }

        // 打开命令执行：模拟打开文件
        private void Open_Executed(object sender, ExecutedRoutedEventArgs e)
        {
            txtEditor.Text = "模拟打开的文件内容...";
            txtStatus.Text = "已打开文件";
            e.Handled = true;
        }

        // 保存命令执行：模拟保存
        private void Save_Executed(object sender, ExecutedRoutedEventArgs e)
        {
            txtStatus.Text = "文件已保存";
            e.Handled = true;
        }

        // 保存命令可用性：文本不为空时可执行
        private void Save_CanExecute(object sender, CanExecuteRoutedEventArgs e)
        {
            e.CanExecute = !string.IsNullOrEmpty(txtEditor.Text);
            e.Handled = true;
        }

        // 新建/打开命令始终可执行
        private void AlwaysCanExecute(object sender, CanExecuteRoutedEventArgs e)
        {
            e.CanExecute = true;
            e.Handled = true;
        }
    }
}
```

### **运行效果**

1. **UI 自动显示文本**：菜单和工具栏按钮的文本会自动使用 `RoutedUICommand.Text` 属性（如 “新建”“打开”），无需手动设置 `Header` 或 `Content`。
2. **快捷键支持**：按 `Ctrl+N`（新建）、`Ctrl+O`（打开）、`Ctrl+S`（保存）可触发对应命令。
3. **状态自动控制**：
    - 文本为空时，“保存” 按钮自动禁用（`CanExecute` 返回 `false`）。
    - 输入文本后，“保存” 按钮启用。

### **核心特性体现**

- **`Text` 属性的便捷性**：命令文本只需定义一次，菜单、按钮等控件可通过绑定自动同步，避免硬编码重复。
- **继承路由能力**：与 `RoutedCommand` 一样支持元素树路由，命令源（菜单 / 快捷键）无需关心执行逻辑的位置。
- **标准化命令**：适合定义需要在多处 UI 展示的命令（如系统级操作 “复制”“粘贴”），保持界面一致性。

### 9.1.4命令库
**`WPF`提供了基本命令库，基本命令库中保存有超过百余条。并且这些命名有以下5个专门的静态类的静态属性提供：**

- **ApplicationCommands:** 该类提供了通用命令，包括剪贴板命令（如Copy、Cut、和Save)以及文档命令（如New、Open、Save、Save As、Print等）。
- **NavigationCommands:** 该类提供用于导航的命令，包括为基于也页面的应用程序设计的一些命令（如BrowseBack、BrowseForward和NextPage),以及其他适合于基于文档的应用程序的命令（如IncreseZoom和Refresh)。
- **EditingCommands:** 该类提供了许多重要的文档编辑命令，包括用于移动的命令（MoveToLineEnd、MoveLeftByWord、MoveUpByPage)，选择内容的命令（SelectToLineEnd、SelectLeftByWord），以及改变格式的命令（ToggleBold和ToggleUnderLine）。
- **ComponentCommands:** 该类提供了由用户界面组件使用的命令，包括用于移动和选择内容的命令，这些命令和EditingCommands类中的一些命令类似。
- **MediaCommands:** 该类提供了一组用于处理多媒体的命令（如Play、Pause、NextTrack以及IncreaseVolume）。
以下提供一个简单的示例：
#### **步骤 1：在 XAML 中绑定命令源**

将命令绑定到按钮、菜单项等控件（命令源），无需手动定义命令，直接引用内置命令。

xaml

```xaml
<Window x:Class="BuiltInCommandsDemo.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="WPF内置命令示例" Height="300" Width="400">
    
    <StackPanel Margin="10">
        <!-- 绑定ApplicationCommands的复制、粘贴命令 -->
        <Button Content="复制 (Ctrl+C)" Command="ApplicationCommands.Copy" />
        <Button Content="粘贴 (Ctrl+V)" Command="ApplicationCommands.Paste" />
        
        <!-- 文本框（命令目标：复制/粘贴的作用对象） -->
        <TextBox x:Name="txtContent" Margin="5" Height="100" AcceptsReturn="True" />
    </StackPanel>
</Window>
```

#### **步骤 2：添加命令绑定（CommandBinding）**

内置命令仅定义了 “命令标识”，需通过 `CommandBinding` 关联具体执行逻辑（`Executed`）和可用性检查（`CanExecute`）。

xaml

```xaml
<Window ...>
    <!-- 为ApplicationCommands.Copy和Paste添加绑定 -->
    <Window.CommandBindings>
        <!-- 复制命令绑定 -->
        <CommandBinding 
            Command="ApplicationCommands.Copy"
            Executed="Copy_Executed"
            CanExecute="Copy_CanExecute"
        />
        <!-- 粘贴命令绑定 -->
        <CommandBinding 
            Command="ApplicationCommands.Paste"
            Executed="Paste_Executed"
            CanExecute="Paste_CanExecute"
        />
    </Window.CommandBindings>

    <!-- 界面元素同上 -->
</Window>
```

#### **步骤 3：实现命令逻辑**

在后台代码中编写 `Executed`（执行操作）和 `CanExecute`（检查可用性）事件。

csharp

```csharp
using System.Windows;
using System.Windows.Input;

namespace BuiltInCommandsDemo
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();
        }

        // 复制命令执行：将选中文本复制到剪贴板
        private void Copy_Executed(object sender, ExecutedRoutedEventArgs e)
        {
            if (!string.IsNullOrEmpty(txtContent.SelectedText))
            {
                Clipboard.SetText(txtContent.SelectedText);
            }
            e.Handled = true;
        }

        // 复制命令可用性：有选中文本时可执行
        private void Copy_CanExecute(object sender, CanExecuteRoutedEventArgs e)
        {
            e.CanExecute = !string.IsNullOrEmpty(txtContent.SelectedText);
            e.Handled = true;
        }

        // 粘贴命令执行：从剪贴板粘贴文本
        private void Paste_Executed(object sender, ExecutedRoutedEventArgs e)
        {
            if (Clipboard.ContainsText())
            {
                txtContent.Paste(); // 直接使用TextBox的Paste方法
            }
            e.Handled = true;
        }

        // 粘贴命令可用性：剪贴板有文本且文本框可编辑时可执行
        private void Paste_CanExecute(object sender, CanExecuteRoutedEventArgs e)
        {
            e.CanExecute = Clipboard.ContainsText() && txtContent.IsEnabled;
            e.Handled = true;
        }
    }
}
```

#### **步骤 4：（可选）指定命令目标**

若命令源与目标（如文本框）分离，可通过 `CommandTarget` 指定命令作用的元素：

xaml

```xaml
<!-- 按钮在另一个容器中，指定目标为文本框 -->
<Button 
    Content="复制" 
    Command="ApplicationCommands.Copy" 
    CommandTarget="{Binding ElementName=txtContent}" 
/>
```

### **3. 关键特性**

- **自带快捷键**：内置命令已绑定默认快捷键（如 `ApplicationCommands.Copy` 对应 `Ctrl+C`），无需额外处理键盘事件。
- **自动同步文本**：作为 `RoutedUICommand`，其 `Text` 属性可直接用于菜单或按钮的显示文本，例如：
    
    xaml
    
    ```xaml
    <MenuItem Command="ApplicationCommands.Save" /> <!-- 自动显示“保存” -->
    ```
    
- **路由机制**：命令会沿元素树冒泡，自动寻找对应的 `CommandBinding`，适合复杂 UI 层级。

### **4. 示例扩展：使用 NavigationCommands**

以下是导航命令的简单示例（模拟浏览器前进 / 后退）：

xaml

后台代码：

csharp

```csharp
private int _navIndex = 0;

private void GoBack_Executed(object sender, ExecutedRoutedEventArgs e)
{
    _navIndex--;
    txtStatus.Text = $"当前位置：{_navIndex}";
}

private void CanGoBack(object sender, CanExecuteRoutedEventArgs e)
{
    e.CanExecute = _navIndex > 0; // 索引>0时可后退
}

// 前进命令逻辑类似（略）
```

### **总结**

使用 WPF 内置命令库的核心步骤是：

1. 绑定命令到 UI 元素（命令源）；
2. 通过 `CommandBinding` 关联执行逻辑和可用性检查；
3. 利用命令自带的快捷键和文本属性简化开发。

## 9.2执行命令
### 9.2.1命令源
命令库中的命令始终可用。触发它们的最简单的方法是将它们关联到实现了`ICommandSource`接口的控件，其中包括继承自`ButtonBase`类的控件（Button和CheckBx等）、单独的ListBoxItem对象、Hyperlink以及MenuItem。

**ICommandSource接口定义了三个属性：**

| 名称               | 说明                |
| :--------------- | ----------------- |
| Command          | 指向连接的命令，这是唯一必须的细节 |
| CommandParameter | 提供其他希望随命令发送的数据    |
| CommandTaget     | 确定将在其中执行命令的元素     |

==个人总结：MenuItem控件类具备智能的设计，它可用自动从命令中提出文本并将其设定在Header属性上，同时还会提出Command.InoutBindings集合中的第一个快捷键，显示在文本的旁白==。同时可通过微调命令文本，让其他实现了ICommandSource接口的控件也具备该项功能，其实现的价值体现于使用不同语言本地化应用程序时，可实现灵活的切换语言。
简单示例：
```XAML

<Button Margin="5" Padding="5" Command="ApplicaitonCommands.New" Content="{Binding RelativeSource={RelativeSource Self},Path=Command.Text}"/>
```

同时提供一种直接调用命令的方法：(调用命令的Execute()方法)
```csharp
ApplicationCommands.New.Execute(null,targetElement);
```

### 9.2.3具有内置命令的控件
一些输入控件可自行处理命令事件。例如，TextBox类处理Cut、Copy、以及Pase命令，甚至一些来自于EdintingCommand类的用于选择文本以及将光标移到不同位置的命令。

在此将重点以ToolBar类提供一些内置逻辑，可将其子元素的CommandTarget属性动态的设置为当前具有焦点的控件。若其他控件也需要类似的功能时，可将其添加FocusManager.IsFocusScope创建新的焦点范围。当触发命令时，该焦点范围会通知WPF在父元素的焦点范围内查找元素。ToolBar和Menu控件默认将FocusManager.IsFocusScope设为True。
```XAML
<StackPanel FocusManager.IsFocusScope="True">
  <Button Command="Cut">Cut</Button>
```
在场提供一种使用InputBinding集合删除触发命令的输入的方法。例如可用代码禁用TextBox控件中Copy命令的Ctrl+C的组合键。

```csharp
KeyBinding keyBinding=new KeyBinding(ApplicationCommands.NotACommand,Ket.C,ModifierKeys.Control);
txt.InputBindings.Add(keyBinding);
```

## 9.3高级命令
### 9.3.1自定义命令
在5个命令类(`ApplicationCommands、NavigationCommands、EditingCommands、CommponentCommands、MediaCommands `）中储存命令，显然不会未应用程序提供所有可能的命令需求。可通过实例化一个新的RoutedUICommands对象。

```csharp
public class DataCommands
{
  Private Static RoutedUiCommand requery;
  
  Static DataCommands()
  {
    //实例化一个输入手势化集合InputGestureCollection
    InputGestureCollection inputs=new InputGestureCollection();
    inputs.Add(new KeyGesture(Key.R,ModifierKeys.Control,"Ctrl+R"));
    requery=new RoutedUICommand("Requery","Requery",typeof(DataCommands),inputs);
  }
  Public Static RoutedUICommand Requery
  {
    get{return requery;}
  }
}
```
==细节提示：在开发过程中，我们可以通过调用`CommandManager.InvalidateRequerySuggested()`方法，通知WPF程序评估命令状态，随后即会触发CanExecute事件，并更新使用该命令的任意命令源。==
### 9.3.2使用命令参数
在使用命令过程中，有时候我们需要传递额外的信息，在这时我们可以设置`CommandsParameter`属性来设定传递的信息。
```XAML
<Button Command="NavigationCommands.Zomm" CommandParameter="{Binding ElementName=txtZomm，Path="Text"} Content="Zoom To Value"/>
```
