# 形状、画刷和变换
在`WPF`用户界面中，绘制2D图形内容的最简单的方法时使用形状（Shape）——专门用于表示简单的直线、椭圆、矩形、以及多边形的一些类。从技术的角度来看，形状就是所谓的绘图图元（primitive)。可组合着这些基本元素来创建更复杂的图形。
关于WPF中形状的最重要的细节是，它们都继承自FrameworkElement类。因此，形状是元素。
- **形状绘制自身**。不需要管理无效的情况和绘图过程。例如：当移动内容、改变窗口尺寸或改变形状属性时，不需要手动重新绘制形状。
- **使用与其他元素相同的方式组织形状**。利用布局控件来布置形状。
- **形状支持与其他元素相同的事件**。这意味着形状为了处理焦点、按下键盘、移动鼠标以及单击鼠标等，不必执行任何额外工作。
## 12.1Shape类
每个形状都继承自Shape。其定义的属性有：

|                          名称                          | 说明                                                                           |
| :--------------------------------------------------: | :--------------------------------------------------------------------------- |
|                         Fill                         | 设置绘制表面（边框内的所有内容）的画刷对象                                                        |
|                        Storke                        | 设置绘制形状边缘（边框）的画刷对象                                                            |
|                   StorkeThickness                    | 用设备无关单位设置边框宽度                                                                |
|        StorkeStartLineCap<br>StorkeEndLineCap        | 决定直线开始端和结束端边缘的轮廓。这些属性只影响Line、Polyline以及（在某些状况下）Path形状。所有其他形状都是闭合的，没有开始点和技术点。 |
| StorkeDashArray<br>StorkeDashOffset<br>StorkeDashCap | 用于在形状周围创建点划线边框。可控制点划线的尺寸和频率，以及每条点划线开始端和技术端边缘的轮廓。                             |
|          StrokeLineJoin<br>StrokeMiterLimit          | 确定形状拐角处的轮廓。从技术角度看，这些属性不影响不懂直线相遇的顶点，如矩形的拐点。对于没有拐角的形状，不起效。                     |
|                       Stretch                        | 确定形状如何填充可用的区域。可使用该属性创建能够扩展以适合其容器的形状。还可以为水平对齐和垂直对齐属性使用Stretsh值，强制形状在某个方向上扩展。  |
|                   DefininfGeometry                   | 为形状提供Geometry对象。Geometry对象描述了形状的坐标和尺寸，不包括UIElement类相关的内容。                    |
|                  GeometryTransform                   | 可通过该属性Transform对象，改变用于绘制形状的坐标系统，从而可扭曲、选转、或者移动形状。                             |
|                   RenderedGeometry                   | 提供描述最终的、以渲染好的图形的Geometry对象                                                   |
### 12.1.2矩形和椭圆
矩形和椭圆时两个最简单的形状。为创建矩形或椭圆，需要设置大家属性的Height和Width属性来定义形状的尺寸。
示例：
```XAML

<StackPanel>
  <Ellipse Fill="Yellow" Stroke="Blue" Height="50" Width="100" Margin="5"
                     HorizontalAlignment="Left"/>
  <Rectangle Fill="Yellow" Stroke="Blue"  Height="50" Width="100" Margin="5"
                       HorizontalAlignment="Left"/>
</StackPanel>
```
==Ellipse类没有增加任何属性。Rectangle类只添加了两个属性：RadiusX和RadiusY。其作用时用于填充矩形拐角的椭圆。==
### 12.1.3改变形状的尺寸和放置形状
改变形状尺寸的行为依赖于Stretch属性的值。默认情况下，该值属性被设置为Fill。如果没有设定明确的尺寸，这一设置回拉伸形状，使其填满容器。

|      名称       |                        说明                         |
| :-----------: | :-----------------------------------------------: |
|     Fill      |              形状拉伸其宽度和高度，从而可以正好适应其容器               |
|     None      | 形状不被拉伸。除非使用Height和Width属性将形状的高度和宽度设置为非0值，否则形状不会显示 |
|    Uniform    |  按比例改变形状的高度和宽度，直至形状到达容器边缘。如果为椭圆设置该值，会得到适应窗口最大的圆。  |
| UniformToFill |         按比例改变形状的宽度和高度，直到形状填充满了整个可用的高度和宽度。         |
### 12.1.4使用Viewbox控件缩放形状
ViewBox时继承自Decorator的简单类。该类只接受一个子元素，并拉伸或缩小子元素以适应可用的控件。默认是Uniform等比例缩放。再而可通过使用StretchDirection属性获得更大的控制权。默认情况下该值为Both，但使用UpOnly值创建只会增长而不会收缩超过原始尺寸的内容，使用DownOnly与前者相反。以下提供一个简单的示例：

```XAML

<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="auto"/>
        <RowDefinition Height="*"/>
    </Grid.RowDefinitions>
    <TextBlock Text="The First row of a Grid"/>
    <Viewbox Grid.Row="1" HorizontalAlignment="Left">
        <Canvas Width="200" Height="150">
            <Ellipse Fill="Yellow" Stroke="Blue" Canvas.Left="10" Canvas.Top="50"
                     Width="100" Height="50" HorizontalAlignment="Left"/>
            <Rectangle Fill="red" Stroke="Blue" Canvas.Left="30" Canvas.Top="40"
                       Width="100" Height="50" HorizontalAlignment="Left"/>
        </Canvas>
    </Viewbox>
</Grid>
```
==注意：在Viewbox中放置布局容器时，需要指定其高宽，不然Viewbox无法知道按多少比例进行伸缩。==
### 12.1.5直线
Line形状表示连接一个点和另一个点的一条直线。起点和终点由四个属性设置：X1与Y1表示第一个点，X2与Y2表示第二个点。在Line模型中，Fill、Margin、HorizontalAlignment、VerticalAlignment属性设置会变得没有意义。
示例：
```XAML
<Line Storke="Blue" X1="0" Y1="0" X2="10" Y2="100"/>
```

### 12.1.6折线
可通过PolyLine类绘制一系列的相互连接的直线，只需要使用Points属性提供一系列的坐标点即可。从技术上看Points需要使用PointCollection对象。
示例：
```XAML
<PolyLine Stroke="Blue" Points="0,0 5,100 15,200"/>
```
再提供一个后台代码的数据折线实现方法：
```XAML

<Grid Margin="20">
    <Grid.RowDefinitions>
        <RowDefinition Height="*"/>
        <RowDefinition Height="Auto"/>
    </Grid.RowDefinitions>

    <!-- 用于绘制折线的画布 -->
    <Canvas x:Name="drawingCanvas" 
        Background="#F5F5F5" >
        <!-- 折线控件 -->
        <Polyline x:Name="randomPolyline" 
              Stroke="Blue" 
              StrokeThickness="2" 
              StrokeDashCap="Round"
              StrokeStartLineCap="Round"
              StrokeEndLineCap="Round"/>
    </Canvas>

    <!-- 控制按钮 -->
    <StackPanel Grid.Row="1" Orientation="Horizontal" Margin="0,10,0,0" >
        <Button x:Name="startBtn" Content="开始" Click="StartBtn_Click" Width="75"/>
        <Button x:Name="stopBtn" Content="停止" Click="StopBtn_Click" Width="75" IsEnabled="False"/>
        <Button x:Name="clearBtn" Content="清除" Click="ClearBtn_Click" Width="75"/>
    </StackPanel>
</Grid>
```

```csharp
    public partial class MainWindow : Window
    {

        // 定时器用于控制折线更新频率
        private DispatcherTimer _timer;
        // 随机数生成器
        private Random _random;
        // 存储折线的点集合
        private PointCollection _points;
        // 最大点数限制，防止性能下降
        private const int MaxPoints = 50;
        // 上一个点的X坐标，用于保证折线连续
        private double _lastX = 0;

        public MainWindow()
        {
            InitializeComponent();
            InitializeComponents();
        }

        private void InitializeComponents()
        {
            _random = new Random();
            _points = new PointCollection();
            randomPolyline.Points = _points;

            // 初始化定时器，每500毫秒更新一次
            _timer = new DispatcherTimer();
            _timer.Interval = TimeSpan.FromMilliseconds(500);
            _timer.Tick += Timer_Tick;
        }

        private void Window_Loaded(object sender, RoutedEventArgs e)
        {
            // 窗口加载后确保获取到正确的画布尺寸
        }

        private void Timer_Tick(object sender, EventArgs e)
        {
            // 生成新的随机点并添加到折线
            AddRandomPoint();
        }

        private void AddRandomPoint()
        {
            // 获取画布可用区域尺寸
            double canvasWidth = drawingCanvas.ActualWidth;
            double canvasHeight = drawingCanvas.ActualHeight;

            // 确保画布有有效尺寸
            if (canvasWidth <= 0 || canvasHeight <= 0)
                return;

            // 计算X坐标，每次增加一个小的固定值，使折线从左到右绘制
            _lastX += 5;

            // 当折线到达右边界时，清除现有点重新开始
            if (_lastX > canvasWidth)
            {
                _points.Clear();
                _lastX = 0;
                return;
            }

            // 生成随机Y坐标（在画布高度范围内）
            double y = _random.NextDouble() * canvasHeight;

            // 添加新点到集合
            _points.Add(new Point(_lastX, y));

            // 限制最大点数，保持性能
            if (_points.Count > MaxPoints)
            {
                _points.RemoveAt(0);
            }
        }

        private void StartBtn_Click(object sender, RoutedEventArgs e)
        {
            _timer.Start();
            startBtn.IsEnabled = false;
            stopBtn.IsEnabled = true;
        }

        private void StopBtn_Click(object sender, RoutedEventArgs e)
        {
            _timer.Stop();
            startBtn.IsEnabled = true;
            stopBtn.IsEnabled = false;
        }

        private void ClearBtn_Click(object sender, RoutedEventArgs e)
        {
            _points.Clear();
            _lastX = 0;
        }
    }
```
### 12.1.7多边形
多边形Polygon类与Polyline相同。只不过Polygon会将最后一个点和第一个点连接起来形成封闭的集合图形。同时，每个Polygon和Polyline形状都有FillRule属性，该属性用于从两种填充方法中选择一种来填充区域。默认情况下，FillRule属性被设置为EvenOdd,为确定是否填充区域，WPF计算为了到达形状的外部必须穿过的直线数量。如果时奇数则填充，反之亦然。二另一种填充模式Nonzero更加复杂，在考虑穿过直线的数量同时，还会考虑经过每条直线的方向。当在某个方向上直线的数量等于反方向上的直线数量时，就不会填充区域。

### 12.1.8直线线帽和直线交点
当绘制Line和PolyLine形状是，可使用StrokeStartLineCap和StrokeEndLineCap属性选中如何绘制直线的开始端和结束端。其参数枚举如下：

|    属性    | 说明              | 特点         |
| :------: | :-------------- | :--------- |
|   Flat   | 直线在它的最后坐标处立即终止  | /          |
|  Round   | 平滑的绘制拐角         | 会额外增加直线的长度 |
| Triangle | 绘制直线的两条侧边最后交于一点 | 会额外增加直线的长度 |
|  Square  | 使直线端点具有尖锐的边缘    | 会额外增加直线的长度 |

同时除了Line形状外，所有形状都允许使用StrokeLineJoin属性扭曲它们的拐点，其参数枚举如下：

|  属性   | 说明      |
| :---: | :------ |
| Miter | 使用尖锐的边缘 |
| Bevel | 切掉点边缘   |
| Round | 平滑的过渡边缘 |

当为较宽并且角度非常小的直线拐点使用尖锐的边缘时，尖锐的拐点会不切实际的延伸一段距离。对于这种情况可以使用Bevel或者Round设置修减拐角。也可以使用StrokeMiterLimit属性，当达到特定长度时，该属性自动地剪切边缘。剪切的距离为数值的1/2倍。

### 12.1.9点划线
可通过设置StrokeDashArray属性来选择实线段长度和空白长度，该选择模式默认从第一个开始实白交替。同时可通过设置StrokeDashOffset属性来控制从哪一个StrokeDashArray值开始。为了美观可使用StrokeDashCap属性来控制边缘的形状。
```XAML

<Polyline Stroke="Blue" StrokeThickness="14" StrokeDashArray="1 2" Points="10 30 60 0 90 40 120 10 350 10" StrokeDashOffset="1" StrokeDashCap="Round"/>
```
## 12.2画刷
画刷填充区域，不管是元素的背景色、前景色还是形状的内部填充和笔画（storke)。最简单的画刷类型是SolidColorBrush，这种画刷填充一种固定、连续的颜色。
常见的画刷类：

|          名称          | 说明                                                                        |
| :------------------: | :------------------------------------------------------------------------ |
|   SolidColorBrush    | 使用单一的连续颜色绘制区域                                                             |
| LinearGradientBrush  | 使用渐变填充绘制区域，渐变的阴影填充从一种颜色变化到另一种颜色                                           |
| RadioalGradientBrush | 使用径向渐变填充绘制区域，除了是在图形模式中从中心点向外部辐射渐变之外，这种画刷和线性画刷类似                           |
|      ImageBrush      | 使用可拉伸、缩放或平铺的图像绘制区域                                                        |
|     DrawingBrush     | 使用Drawing对象绘制区域，该对象可以包含已经定义的形状和位图                                         |
|     VisualBrush      | 使用Visual对象绘制区域。因为所有WPF元素都继承自Visual类，，所以可使用画刷将部分用户界面复制到另一个区域。              |
|   BitmapCacheBrush   | 使用从Visual对象缓存的内容绘制区域。这种画刷和VisualBrush类似，但如果需要在多个地方重用图形或者频繁的绘制图形内容，这种画刷更高效 |
### 12.2.1LinearGradientBrush画刷
可通过LinearGradientBrush画刷出个创建从一种颜色变化到另一种颜色的混合填充。
示例：
```XAML
<Rectangle Width="150" Height="100">
  <Rectangle.Fill>
    <LinearGradientBrush>
      <GradientStop Color="Blue" Offset="0"/>
      <GradientStop Color="White" Offset="1"/>
    </LinearGradientBrush>
  </rectanle.Fill>
</Rectangle>
```
如果需要控制颜色的偏移方向，可使用StartPoint和EndPoint属性控制这些细节。LinearGradientBrush将点（0，0）指定为开始点，（1，1）指定为结束点，而不管该区域实际大小。

```XAML
//从上而下的渐变颜色
<Rectangle Width="150" Height="100">
  <Rectangle.Fill>
    <LinearGradientBrush StartPoint="0,0" EndPoint="0,1">
      <GradientStop Color="Blue" Offset="0"/>
      <GradientStop Color="White" Offset="1"/>
    </LinearGradientBrush>
  </rectanle.Fill>
</Rectangle>
```
### 12.2.2RadialGradientBrush画刷
为指定第一种颜色在渐变中的开始点，需要使用GradientOrigin属性。默认情况下，渐变的开始点（0.5，0.5），该点表示填充区域的中心。

示例：
```XAML
//RadiusY:简单理解为渐变高度
//RadiusX：简单理解为渐变宽度
<Ellipse Margin="5" Storke="Black" StorkeThickness="1" Width="200" Height="200">
  <Ellipse.Fill>
    <RadialGradientBrush RadiusX="1" RadiusY="1" GradientOrigin="0.7,0.3">
      <GradientStop Color="White" Offset="0"/>
      <GradientStop Color="Blue" Offset="1"/>
    </RadialGradientBrush>
  </Ellipse.Fill>
</Ellipse>
```
### 12.2.3ImageBrush画刷
可通过ImageBrush画刷使用位图填充区域。可使用的最常见的文件类型有BMP、PNG、GIF、JPEG文件。可通过设置ImageSource属性来指明希望使用的图像。
示例：
```XAML
ViewBox:剪切图片内容用于填充
<Grid>
  <Grid.Background>
    <IamgeBrush ImageSource="logo.jpg" ViewBox="0.4,0.5,0.2,0.2" />
  </grid.Background>
</Grid>
```

ImageBrush画刷还可以使用表买平铺图像来实现一些有趣的效果。
当平铺图像时，有两种选择：
- 按比例平铺：填充区域始终具有相同数量的平铺图像。为适应填充区域，平铺的图像会扩展或收缩。
- 按固定尺寸平铺：平铺图形始终具有相同的尺寸。填充区域的尺寸决定了显示的平铺图像的数量。
为了使用平铺图像，需要设置ImageSource属性以及Viewport、ViewportUnits、与TileMode属性。示例：

```XAML
//TileMode:平铺的模式选择
//Viewport:平铺图像的尺寸
//Viewportunits:平铺图形的尺寸选择模式
<ImageBrush ImageSource="logo.jpg" TileMode="Tile" Viewport="0,0 0.5,0.5" ViewportUnits="RelativeToBoundingBox"/>
```

|   名称   | 说明                        |
| :----: | :------------------------ |
|  Tile  | 在可用区域复制图像                 |
| FlipX  | 复制图像，但垂直翻转每个第二列           |
| FlipY  | 复制图像，但水平翻转每个第二行           |
| FilpXY | 复制图像，但垂直翻转每个第二列，水平翻转每个第二行 |

## 12.3变换
通过使用变换（transfrom)，许多绘图任务将更趋简单；变换是通过不加通告地切换形状或者元素使用的坐标系统来改变形状或元素绘制的方式的对象。

|         名称         | 说明                                    | 重要属性                          |
| :----------------: | :------------------------------------ | ----------------------------- |
| TranslateTransform | 将坐标系统移动一定距离。如果希望在不同地方绘制相同的形状，该变换非常有用。 | X、Y                           |
|  RotateTransform   | 旋转坐标系统。正常绘制的形状绕着选择的中心旋转               | Angle、CenterX、CenterY         |
|   ScaleTransform   | 放大或者缩小坐标系统，从而绘制更大或者更小的图形              | ScaleX、ScaleY、CenterX、CenterY |
|   SkewTransform    | 通过倾斜一定的角度扭曲坐标系统。例如：将正方形变换为平行四边形       | AngleX、AngleY、CenterX、CenterY |
|  MatrixTransform   | 使用提供的矩阵的乘积修改坐标系统                      | Matrix                        |
|   TansformGroup    | 组合多个变换，从而可以一次应用所有这些变换                 | N/A                           |
### 12.3.1变换形状
为了变换形状，将RenderTransform属性指定为希望使用的变换对象。
示例：
```XAML
//RenderTransformOrigin：变换相对中心
//Angle：旋转角度
<Rectangle Width="80" Height="10" Stroke="Blue" Fill="Yellow" Canvas.Left="100" RenderTransformOrigin="0.5,0.5" Canvas.Top="100">
   <Rectangle.RenderTransform>
      <RotateTransform Angle="25"/>
   </Rectangle.RenderTransform>
</Rectangle>
```
## 12.4透明
通过配置元素的Opcity属性来控制控件的透明度。
### 12.4.1透明掩码
Opacity属性使元素的所以内容都是部分透明的。OpacityMask属性提供了更强大的灵活性。可使元素的特定区域透明或者部分透明，从而实现各种常见的以及新颖的效果。
例如：
```XMAL

 <Button FontSize="14" FontWeight="Black" Height="50" Width="120">
     <Button.OpacityMask>
         <LinearGradientBrush StartPoint="0,0" EndPoint="1,0">
             <GradientStop Offset="0" Color="Black"/>
             <GradientStop Offset="1" Color="Transparent"/>
         </LinearGradientBrush>
     </Button.OpacityMask>
     <Button.Content>A Partially Transparent Button</Button.Content>
 </Button>
```
