# 格式化绑定的数据
## 20.2数据转换
WPF提供了两种方式便于我们在数据绑定的过程中对数据进行转行，已达到我们需要的效果：
- **字符串格式化:** 使用该功能可以通过设置**Binding.StringFormat** 属性对文本形式的数据进行转换——例如包含日期和数值的字符串。对于至少一半的格式化任务，字符串格式化是一种便捷的技术。
- **值转换器：** 该功能更强大（有时更复杂），使用该功能可将任意类型的源数据转换为任意类型的对象表示，然后可传递到关联的控件。
### 20.2.1使用StringForamt属性
当使用Binding.StringFormat属性时，使用标准的.Net格式字符串，具体形式{0：C}。其中0代表第一个数值，C引用希望应用的格式化字符串。
示例：
```XAML

 <TextBox Margin="5" Text="{Binding Path=Proce,StringFormat={}{0:C}}"></TextBox>
 <TextBox Margin="5" Text="{Binding Path=Proce,StringFormat=The value is {0:C}.}"></TextBox>
  <Button Margin="5" Content="切换数值" Click="Button_Click"/>

```

```Csharp

  private int proce;
  public int Proce
  {
      get
      {
          return proce;
      }
      set 
      { 
      proce=value; PropertyChanged?.Invoke(this, new PropertyChangedEventArgs("Proce"));
      }
      
  private void Button_Click(object sender, RoutedEventArgs e)
  {
      Proce = new Random().Next(1, 100);
  }
```
==注意：StringFormat属性的值以括号{}开头。完整值是{}{0：C}而不是{0：C}。为了转义字符串，需要在开始处使用稍微笨拙的花括号对。同时只有当StringForamt值义花括号开头时才需要转义==

### 20.2.2值转换器简介
值转换器是WPF绑定难题中非常有用的一部分。可通过如下几种有用的方式使用它们：
- **将数据格式化为字符串表示形式:** 例如，可将数字转换为货币字符串。
- **创建特定类型的WPF对象：** 例如，可读取一块二进制数据，并在创建一副能绑定到Image元素的BitmapImage对象。
- **根据绑定数据又条件地改变元素中的属性：** 例如，创建值转换器，用于改变元素的背景色以突出显示位于特定范围内的数值。
### 20.2.3使用值转换器设置字符串的格式
为创建值转换器，需要执行以下4个步骤：
- 创建一个实现了IValueConverter接口的类
- 为该类声明添加ValueConversion特性，并指定目标数据类型
- 实现Convert()方法，该方法将数据从原来的格式转换为显示的格式
- 实现ConvertBack()方法，该方法执行反向变换，将值从显示格式转换为原格式。
示例:
```XAML
// ConverterCulture="en-US"值转换器使用的文化配置
<TextBox Margin="5">
    <Binding Path="Proce" Mode="TwoWay" ConverterCulture="en-US">
        <Binding.Converter>
            <loc:PriceConverter></loc:PriceConverter>
        </Binding.Converter>
    </Binding>
</TextBox>
```

```Csharp

//decimal：源数据类型 string:转行目标数据类型
  [ValueConversion(typeof(decimal), typeof(string))] 
  public class PriceConverter : IValueConverter
  {
      //源数据转行显示格式
      public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
      {
         decimal price = (decimal)value;
          return price.ToString("C",culture);
      }

      //显示格式转换为源数据
      public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
      {
          string price=value.ToString();
          decimal result;
          if (Decimal.TryParse(price, NumberStyles.Any, culture, out result))
          {
              return result;
          }
          return value;
      }
  }
```

==注意：应用程序会自动读取当前线程的文化设置，使用为了控制文化信息的使用，应显示设置==

### 20.2.6评估多个属性
对多个目标进行值转换时，WPF提供了MultiBinding对象，将其代替Binding对象即可实现对多个目标进行值转换。
示例：
```XAML

        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
        xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
        xmlns:local="clr-namespace:WPF第20章学习"
        xmlns:converters="clr-namespace:WPF第20章学习.ValueConverters"
        mc:Ignorable="d"
        Title="多属性评估示例" Height="450" Width="800">

    <converters:MultiValueConverter x:Key="StatusRoleBrushConverter"/>
</Window.Resources>
    <StackPanel Margin="20">
        <TextBlock Text="状态："/>
```

## 20.3列表控件
WPF提供了充足的格式化选择，这些格式化选择中的大部分都构建了ItemsControl基类中，所有列表控件都继承自该类，所有应当从该类开始研究列表格式化。ItemsControl类定义支持数据绑定两个重要的格式化属性（样式和数据模板）。

| 名称                         | 说明                                                                                                     |
| :------------------------- | :----------------------------------------------------------------------------------------------------- |
| ItemsSource                | 绑定数据源（希望在列表中显示的集合或DataView对象）                                                                          |
| DisplayMemberPath          | 希望为两个数据想显示的属性。对于更复杂的表达式或者为了使用多个属性的组合而言，应改为ItemTemplate属性                                               |
| ItemStringFormat           | 该属性是.NET格式字符串，如果设置了该属性，将使用该属性为每一个想格式化文本。                                                               |
| ItemContainerStyle         | 该属性是一个样式，通过该样式可以设置封装每个项的容器的多个属性。容器取决于列表的类型。当填充列表时，自动创建这些封装器对象                                          |
| ItemContainerStyleSelector | 使用代码为列表中每项的封装器选择样式的StyleSelector对象。可使用该属性为列表中的不同项提供不同的样式。必须创建自定义的StyleSelector类。                       |
| AlternationCount           | 在数据中设置的交替集合数量。例如：将AlternationCount设置为2，将两个不同的行样式交替。以此类推                                                |
| ItemTemplate               | 此类模板从绑定的对象中提取合适的数据，并将提取的数据安排到合适的控件组合中。                                                                 |
| ItemTemplateSelector       | 使用代码为列表中的每个项选择模板的DataTemplateSelector对象。可通过这个属性为不同的项应用不同的模板。必须创建自定义的DataTemplateSelector类              |
| ItemsPanel                 | 定义用于包含列表中项的面板。所有项封装器都添加到这个容器中。通常使用垂直的VirtualizingStackPanel面板                                          |
| GroupStyle                 | 如果正在使用分组，这个样式定义了应当如何格式化每个组。当使用分组时，项封装器（ListBoxItem和ComboBoxItem)被添加到表示每个分组的GroupItem封装器中，然后这些分组被添加到列表中 |
| GroupStyleSelector         | 使用代码为每个分组选择样式的StyleSelector，必须创建自定义的StyleSelector类。                                                    |

##20.4.2交替条目样式
WPF通过两个属性为交替项提供了内置支持：ALternationCount和AlternationIndex。AlternationCount指定序列中项的数量，经过给该数量的项之后交替样式。默认情况下该属性值为0，而且不使用交替格式。如果将AlternationCount设置为1，列表将在每项之后交替。
示例：
```XAML

<ListBox Margin="5" AlternationCount="2" ItemsSource="{Binding Lists}"  Height="200" Width="200">
    <ListBox.ItemContainerStyle>
        <Style TargetType="{x:Type ListBoxItem}">
            <Setter Property="Background" Value="LightSteelBlue"/>
            <Setter Property="Margin" Value="5"/>
            <Setter Property="Padding" Value="5"/>
            <Style.Triggers>
                <Trigger Property="ItemsControl.AlternationIndex" Value="1">
                    <Setter Property="Background" Value="LightBlue"/>
                </Trigger>
                <Trigger Property="IsSelected" Value="True">
                    <Setter Property="Background" Value="DarkRed"/>
                    <Setter Property="Foreground" Value="White"/>
                    <Setter Property="BorderBrush" Value="Black"/>
                    <Setter Property="BorderThickness" Value="1"/>
                </Trigger>
            </Style.Triggers>
        </Style>
    </ListBox.ItemContainerStyle>
</ListBox>
```

### 20.4.2样式选择器
WPF构建了专门的继承自StyleSelector的类，该类复杂检查每个数据项并选择合适的样式。该工作是在SelectStyle()方法中执行的，必须重写该方法。定义样式选择器，通过判断绑定数据集合中子项的特定属性来甄别选用那种样式。
示例：
```XAML

<Window x:Class="WPF第20章学习.Window3"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
        xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
        xmlns:local="clr-namespace:WPF第20章学习"
        xmlns:loc="clr-namespace:WPF第20章学习.Models"
        mc:Ignorable="d"
        Title="Window3" Height="450" Width="800">
    <Window.Resources>
        <Style x:Key="AdminStyle" TargetType="{x:Type ListBoxItem}">
            <Setter Property="Foreground" Value="Red"/>
            <Setter Property="FontWeight" Value="Bold"/>
            <Setter Property="FontSize" Value="16"/>
        </Style>

        <Style x:Key="Userstyle" TargetType="{x:Type ListBoxItem}">
            <Setter Property="Foreground" Value="Blue"/>
            <Setter Property="FontWeight" Value="Normal"/>
            <Setter Property="FontSize" Value="14"/>
        </Style>

        <loc:UserRoleStyleSelector x:Key="RoleStyleSelector" AdminStyle="{StaticResource AdminStyle}"
                                   UserStyle="{StaticResource Userstyle}">
        </loc:UserRoleStyleSelector>
        
    </Window.Resources>
    <Grid Margin="10">
        <ListBox ItemsSource="{Binding Users}" ItemContainerStyleSelector="{StaticResource RoleStyleSelector}">
            <ListBox.ItemTemplate>
                <DataTemplate>
                    <TextBlock Text="{Binding Name}"/>
                </DataTemplate>
            </ListBox.ItemTemplate>
        </ListBox>
    </Grid>
</Window>
```

**自定义样式选择器**
```Csharp

public class UserRoleStyleSelector : StyleSelector
{
    public Style AdminStyle { get; set; }
    public Style UserStyle { get; set; }

    public override Style SelectStyle(object item, DependencyObject container)
    {
        if (item is User user)
        {
            return user.Role=="Admin"?AdminStyle : UserStyle;
        }
        //默认样式
        return null;
    }
}
```

## 20.5数据模板
数据模板时一块定义如何显示绑定的数据对象的XAML标记。有两种类型的控件支持数据模板：
- 内容控件通过ContentTemplate属性支持数据模板。内容模板用于显示任何放置在Content属性的内容中。
- 列表控件（继承自ItemsControl类的控件）通过ItemTemplate属性支持数据模板。这个数据模板用于显示作为ItemsSource提供的集合中的每个项（或来自DataTable的每一行）。
示例：
```XMAL

//数据模型用上列
<ListBox Grid.Row="1" ItemsSource="{Binding Users}">
    <ListBox.ItemTemplate>
        <DataTemplate>
            <Border Margin="3" BorderThickness="1" BorderBrush="SteelBlue" CornerRadius="4">
                <StackPanel Orientation="Horizontal">
                    <TextBlock Text="{Binding Path=Name}" Margin="2"/>
                    <TextBlock Text="{Binding Path=Role}" Margin="2"/>
                </StackPanel>
            </Border>
        </DataTemplate>
    </ListBox.ItemTemplate>
</ListBox>
```

### 20.5.1分离和重用模板

### 20.5.2改变模板
使用目前为止介绍的数据模板的一个限制是，只能为整个列表使用一个模板。但在许多情况下，使用采用不同方式灵活地表示不同的数据。
- 使用数据触发器：可根据绑定的数据对象种的属性值使用触发器修改模板中的属性。除了不需要的依赖想属性外，数据触发器义第11章中介绍的有关样式的属性触发器的工作方式类似。
- 使用值转换器：实现了IValueConverter接口的类，能够将值从绑定的对象转换为可用于设置模板化中与格式化相关的属性的值。
- 使用模板选择器：模板选择器检查绑定的数据对对象，并在几个不同的模板之间就行选择。

示例1：使用数据触发器
```XAML

 <Window.Resources>
     <DataTemplate x:Key="DefultTemplate">
         <DataTemplate.Triggers>
             <DataTrigger Binding="{Binding Path= Role}" Value="Admin">
                 <Setter Property="ListBoxItem.Foreground" Value="red"/>
             </DataTrigger>
         </DataTemplate.Triggers>
         <Border Margin="5" BorderThickness="1" BorderBrush="SteelBlue" CornerRadius="4">
             <Grid Margin="3">
                 <Grid.RowDefinitions>
                     <RowDefinition/>
                     <RowDefinition/>
                 </Grid.RowDefinitions>
                 <TextBlock FontWeight="Bold" Text="{Binding Path=Name}"/>
                 <TextBlock Grid.Row="1" Text="{Binding Path=Role}"/>
             </Grid>
         </Border>
     </DataTemplate>
 </Window.Resources>
     
 <Grid>
     <Grid.RowDefinitions>
         <RowDefinition/>
         <RowDefinition/>
     </Grid.RowDefinitions>
     <ListBox  ItemsSource="{Binding Users}" ItemTemplate="{StaticResource ResourceKey=DefultTemplate}">
     </ListBox>
 </Grid>
```

### 20.5.3模板选择器
模板选择器的工作方式和在前面分析的样式选择器的工作方式相同——它们检查绑定对象并使用自定义的逻辑选择适当的模板。为此需要创建DataTemplateSelector的类。
示例：
```XAML
 <Window.Resources>
     <DataTemplate x:Key="DefultTemplate">
         <Border Margin="5" BorderThickness="1" BorderBrush="SteelBlue" CornerRadius="4" Background="red">
             <Grid Margin="3">
                 <Grid.RowDefinitions>
                     <RowDefinition/>
                     <RowDefinition/>
                 </Grid.RowDefinitions>
                 <TextBlock FontWeight="Bold" Text="{Binding Path=Name}"/>
                 <TextBlock Grid.Row="1" Text="{Binding Path=Role}"/>
             </Grid>
         </Border>
     </DataTemplate>
     <DataTemplate x:Key="HighlightTemplate">
         <Border Margin="5" BorderThickness="1" BorderBrush="SteelBlue" CornerRadius="4" Background="Orange">
             <Grid Margin="3">
                 <Grid.RowDefinitions>
                     <RowDefinition/>
                     <RowDefinition/>
                 </Grid.RowDefinitions>
                 <TextBlock FontWeight="Bold" Text="{Binding Path=Name}"/>
                 <TextBlock Grid.Row="1" Text="{Binding Path=Role}"/>
             </Grid>
         </Border>
     </DataTemplate>
 </Window.Resources>
     
 <Grid>
     <Grid.RowDefinitions>
         <RowDefinition/>
         <RowDefinition/>
     </Grid.RowDefinitions>
     <ListBox  ItemsSource="{Binding Users}">
         <ListBox.ItemTemplateSelector>
             <loc:UserRoleTemplateSelector DefaultTemplate="{StaticResource DefultTemplate}"
                                           HighlightTemplate="{StaticResource HighlightTemplate}"
                                           PropertyToEvaluate="Role" PropertyValueToHighilight="Admin"/>
         </ListBox.ItemTemplateSelector>
     </ListBox>
 </Grid>
```

```Csharp

    public class UserRoleTemplateSelector:DataTemplateSelector
    {
        public DataTemplate DefaultTemplate
        { 
            get;set;
        }

        public DataTemplate HighlightTemplate
        {
            get;set;
        }

        public string PropertyToEvaluate
        {
            get;set;
        }
        public string PropertyValueToHighilight
        {
            get;set;
        }

        public override DataTemplate SelectTemplate(object item, DependencyObject container)
        {
            if (item!=null)
            {
                //该处使用了反射机制
                User user = item as User;
                Type type = user.GetType();
                PropertyInfo property = type.GetProperty(PropertyToEvaluate);
                if (property.GetValue(user, null).ToString() == PropertyValueToHighilight)
                {
                    return HighlightTemplate;
                }
            }
            return DefaultTemplate;
        }

    }
```
