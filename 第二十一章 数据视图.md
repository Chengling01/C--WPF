# 数据视图
## 21.1View对象
当集合（或DataTable）绑定到ItemsControl控件时，会不加通告地在后台创建数据视图——位于数据源和绑定的控件之间。数据视图是进入数据源的窗口，可以跟踪当前项，并且支持各种功能，如排序、过滤和分组。这些功能和数据对象本身是相互独立的，这意味着可在窗口的不同部分使用不同的方式绑定相同的数据。
使用数据视图对象取决于数据对象的类型。所有视图都继承自CollectionView类，并且有两个继承自CollenctionView类的特殊实现：ListCollectionView和BindingListCollectionView。下面是Collection类的工作原理：
- 如果数据源实现了IBindingList接口，就会创建BindingListCollectionView视图。当绑定到ADO.NET中的DataTable对象时就会创建该视图。
- 如果数据源没有实现IBindingList接口，当实现了IList接口，就会创建ListCollectionView视图。当绑定到ObservableCollection集合（如产品列表）时会创建该视图。
- 如果数据源没有实现IBindingList或IList接口，但实现了IEnumerable接口，就会得到基本的CollectionView视图。

### 21.1.2视图导航
可使用视图对象完成的最简单意见事是确定列表中的项数（通过Count属性），以及获取当前数据对象的引用（通过CurrentItem属性）或当前位置索引(通过CurrentIndex属性)。还可以用少数方法从一条记录移到另一条记录，如MoveCurrentToFirst()、MoveCurrentToLast()、MoveCurrentToNext()、MoveCurrentToPrivious()、MoveCurrentToPosition()。
==注意：代码将视图转换为恰当的视图类型（ListCollectionView),而不使用ICollectionView接口。虽然ICollectionView接口提供了大多数相同的功能，但缺少能提供集合中项数的Count属性==

### 21.2.2过滤DataTable对象
对于DataTable对象，过滤工作是不一同的。每个DataTable对象都与一个DataView对象相关联。与WPF视图一样，可使用DataView对象过滤记录（使用RowFilter属性通过字段内容进行过滤，或者使用RowStateFilter属性通过行状态进行过滤）。DataView对象还通过Sort属性支持排序。与WPF视图对象不同的是，DataView对象不跟踪数据集中的位置。DataView对象还提供了用于锁定编辑能力的附加属性（AllowDelete,AllowEditt以及AllowNew).
可通过检索绑定的DataView对象并直接修改其属性来改变过滤数据列表的方式（通过DataTable.DefaulteView属性获取默认的DataView对象）。
DataTable对象使用BindingListCollectionView视图不自持Filtershuxing1(BindingListCollectionView.CanFilter属性放回false,并如果视图设置Filter属性，就会抛出异常).相反，BindingListCollectionView视图提供了CustomFilter属性。CustomFilter属性本身不能做任何工作——只是接收指定的过滤字符串，并使用这个过滤字符串设置DataView.RowFilter属性。
```Csharp

decimal minimumOPrice;
if(Decimal.TryParse(txtMinPrice.Text,out minimumPrice))
{
  BindingListCollectionView view=CollectionViewSource.GetDefaultView(数据源)
     as BindingListCollectionView;
     if(view is not null)
     {
       //CustomFIlster：字符串过滤器
       view.CustomFilter="UnitCost>"+minimumPrice.ToString();
     }
}
```

### 21.2.3排序
使用System.ComponentModel,SortDescription对象确定希望使用的字段。每个SortDescription对象确定希望用于排序的字段和排序方向（升序和降序）。
示例：
```Csharp

ICollectionView _collectionView;

_collectionView=CollectionViewSource.GetDefaultView(数据源);
//配置排序规则：
_collectionView.SortDescriptions.Add(new SortDescription("ModelName",ListSortDirection.Descending));
```

自定义排序时，只能用于ListCollectionView视图（不能用于BindingListCollectionView视图）。ListCollectionView类提供的CustomSort属性接收一个IComparer对象，IComparer对象在两个数据项之间进行比较，并且指示较大项。
示例：
```Csharp

ListCollectionView view =(ListCollectionView)CollectionViewSource.GetDefaultView(数据源)；
 view.CustomSort=new SortByModelNameLength();
 
 public class SortByModelNameLength : IComparer
{
    public int Compare(object? x, object? y)
    {
        DeviceInfo deviceInfo = x as DeviceInfo;
        DeviceInfo deviceInfo1 = y as DeviceInfo;
        return deviceInfo.DeviceName.Length.CompareTo(deviceInfo1.DeviceName.Length);
    }
}

    public class DeviceInfo
    {
        public string DeviceCode { get; set; }
        public string DeviceName { get; set; }
        public string Status { get; set; }
    }
```

### 21.2.4分组
为执行分组，需要为CollectionView.GroupDescriptions集合添加System.ComponentModel.PropertyGroupDedcription对象。如下简单示例：
```Csharp
//分组标志需要时模型名称
ICollectionView view=CollectionViewSource.GetDefaultView(数据源);
view.GroupDescriptions.Add(new PropertyGroupDescription("分组标志"));
```
当使用分组时，列表为每个分组创建了单独的GroupItem对象，并且为列表添加了这些GroupItem对象。GroupItem是内容控件，所以每个GroupItem对象都包含一个适当的具有实际数据的容器（r如ListBoxItem对象）。显示分组的秘密是格式化GroupItem元素，使其突出显示。
可使用样式为列表中的所有GroupItem对象应用格式。然而，您不仅可能希望调整格式——例如，可能还希望显示分组标题，这就需要使用模板。幸运的是，ItemsControl类通过它的ItemsControl.GroupStyle属性简化了这两项任务，该属性提供了一个GroupStyle对象的集合。疏虽然属性名称中包含Style，但GroupStyle类并不是样式，而是一个简单的包，为配置GroupItem对象封装了一些有用的设置。以下列出GroupStyle类的属性。

|           名称           |                                    说明                                    |
| :--------------------: | :----------------------------------------------------------------------: |
|     ContainerStyle     |                       设置被应用到每个分组生成的GroupItem元素的样式                        |
| ContainerStyleSelector | 不是使用ContainerStyle属性，而是使用ContainerStyleSelector属性提供一个类，该类根据分组选择准备使用的正确样式 |
|     HeaderTemplate     |                          允许用户为了在每个分组开头显示内容而创建模板                          |
| HeaderTemplateSelector |   不是使用HeaderTemplate属性，而是使用HeaderTemplateSelector属性提供一个类，该类根据分组选用正确的模板   |
|         Panel          |     该变用于包含分组的模板。例如，可使用WrapPanel面板而非标准的StackPanel面板，创建从左向右然后向下平铺分组的列表     |
实例：

### 21.2.5实时成型
WPF 中的数据视图实时成型用于解决数据项属性发生变化后，集合在不显式调用 `Refresh()` 的情况下自动完成排序、分组和过滤更新的问题，典型适用于实时监控、可编辑列表、动态筛选和多条件联动展示等高频变化场景，其前提是数据项必须能够通知属性变化（如实现 `INotifyPropertyChanged` 或使用 `DataRowView`），并且通过 `CollectionViewSource` 获取集合视图，在此基础上显式启用 `ICollectionViewLiveShaping` 并声明哪些属性会影响排序、分组或过滤，否则视图不会重新计算；在工程实践中应优先使用实体类配合 `ObservableCollection`，明确配置 LiveProperties 以避免不必要的性能开销，从而实现稳定、高效且无手动刷新的数据展示效果。
示例：
```Csharp

public class Product : INotifyPropertyChanged
{
    private decimal _unitCost;
    private string _category;

    public string ProductName { get; set; }

    public decimal UnitCost
    {
        get => _unitCost;
        set
        {
            if (_unitCost != value)
            {
                _unitCost = value;
                OnPropertyChanged(nameof(UnitCost));
            }
        }
    }

    public string Category
    {
        get => _category;
        set
        {
            if (_category != value)
            {
                _category = value;
                OnPropertyChanged(nameof(Category));
            }
        }
    }

    public event PropertyChangedEventHandler PropertyChanged;
    protected void OnPropertyChanged(string name) =>
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
}


public class MainViewModel
{
    public ObservableCollection<Product> Products { get; }

    public ICollectionView ProductsView { get; }

    public MainViewModel()
    {
        Products = new ObservableCollection<Product>
        {
            new Product { ProductName="A", UnitCost=50, Category="低价" },
            new Product { ProductName="B", UnitCost=150, Category="中价" },
            new Product { ProductName="C", UnitCost=300, Category="高价" }
        };

        ProductsView = CollectionViewSource.GetDefaultView(Products);

        ProductsView.SortDescriptions.Add(
            new SortDescription("UnitCost", ListSortDirection.Ascending));

        ProductsView.GroupDescriptions.Add(
            new PropertyGroupDescription("Category"));

        var live = ProductsView as ICollectionViewLiveShaping;
        if (live != null)
        {
            live.IsLiveSorting = true;
            live.IsLiveGrouping = true;

            live.LiveSortingProperties.Add("UnitCost");
            live.LiveGroupingProperties.Add("Category");
        }
    }
}

```
