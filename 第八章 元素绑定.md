# 8.元素绑定
## 8.1从元素绑定到元素
### 1.XAML元素绑定方法
绑定源：ElementName,Path属性：Path
绑定源的绑定属性需要是依赖属性，依赖属性会在属性值改变时通知绑定目标对象更新绑定数据。 

代码示例：

``` Csharp

//绑定源对象：Slider
 //绑定源对象属性：sliderFontSize
 //绑定源对象属性：Value
 //对象目标：TextBlock
 //目标对象属性：FontSize
     <Slider Name="sliderFontSize" Margin="3" Minimum="1"
             Maximum="40" Value="10" TickFrequecy="1" TickPlacement="TopLeft/>
     <TextBlock Margin="10" Text="Simple Text" Name="lblSampleText"
             FontSize="{Binding ElementName=sliderFontSize,Path=Value}"/>
```
 
### 2.绑定模式
|       名称       | 说明                                                               |
| :------------: | :--------------------------------------------------------------- |
|     OneWay     | 当源属性变化时更新目标属性                                                    |
|    OneTime     | 最初根据源属性设置目标属性，其后的所有改变都会忽略，除非调用BindingExpression.UpdateTarget()方法 |
| OneWayToSource | 与OnWay类型类似，但方向相反。当目标属性发生改变时更新源属性，但目标属性永远你不会更新                    |
|    Default     | 默认的绑定方式，可能时双向也可能时单向                                              |
|     TwoWay     | 当源属性变化更新目标属性，并且当目标属性变化时更新源属性                                     |
```js

代码示例：
 <TextBlock Margin="10" Text="Simple Text" Name="lblSampleText"
            FontSize="{Binding ElementName=sliderFontSize,Path=Value,Mode=TwoWay}"/>
```
### 3.使用代码创建绑定
#### 3.1 上列XAML的代码标记方式：
```csharp

Bingding binding=new Binding();
binding.Source=siliderFontSize;
binding.Path=new PropertyPath("Value");
binding.Mode=BindingMode.TwoWay;
lblSampleText.SetBinding(TextBlock.FontSize,binding);
```
#### 3.2绑定的清除

|       调用方法       | 传入参数                | 说明                                         |
| :--------------: | :------------------ | ------------------------------------------ |
|   ClearBinding   | 传入目标对象及其需要清除的绑定依赖属性 | 清除目标对象上的一个绑定，通过BindingOperations静态类直接调用该方法 |
| ClearAllBindings | 传入目标对象              | 清除目标对象上的所有绑定，通过BindingOperations静态类直接调用该方法 |
#### 3.3使用代码来检索绑定
3.3.1通过使用BindingOperations.GetBinding()来检索相应的绑定对象，该方法需要提供两个参数：绑定元素以及具有绑定表达式的属性。
示例：
XAML标记：
```XAML
<TextBlock Margin="10" Text="Simple Text" Name="lblSampleText" FontSize="{Binding ElementName=sliderFontSize,Path=Value}"/>
```

后端代码：
```Csharp

//通过该方法获取lblSampleText对象的FontSize属性的绑定源
Binding binding=BindingOperations.GetBinding(lblSampleText,TextBlock.FontSize);
```
Expression：表达式  Operation:操作
3.3.2在 WPF 中，`BindingExpression` 是数据绑定机制的核心组成部分，它代表**一个具体的数据绑定实例**，负责在绑定源（Source）和绑定目标（Target）之间建立实际的连接并处理数据的传递逻辑。其意义和作用如下：

| 实际执行数据同步        | `Binding` 仅定义绑定的配置（如 `Path="Name"`、`Mode=TwoWay`），而 `BindingExpression` 会根据这些配置，实际监听源对象的属性变化（通过 `INotifyPropertyChanged`）和目标对象的属性变化（通过 `DependencyProperty`），并在两端之间同步数据。 |
| :-------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **存储绑定的运行时状态**  | 源对象和目标对象的实际引用；<br>当前绑定的值（可能经过转换器处理后的值）；<br>绑定是否处于活动状态、是否有错误等。                                                                                                            |
| **提供手动控制绑定的方法** | `UpdateSource()`：强制将目标的值同步到源（常用于`Mode=OneWayToSource` 或需要手动触发更新的场景）；<br> `UpdateTarget()`：强制将源的值同步到目标；<br>`GetHashCode()`、`ToString()` 等方法用于标识和调试绑定实例。                   |
| **处理绑定错误**      | 当绑定过程中出现错误（如路径无效、类型转换失败），`BindingExpression` 会通过 `HasError`、`Errors` 等属性暴露错误信息，便于调试和异常处理。                                                                                |
获取绑定表达式对象示例：
```Csharp
BingdingExpression expression=BindingOperations.GetBindingExpression(目标对象，目标对象的绑定参数);
```
### 4.绑定更新
**细节提示：在绑定模式为OneWay和TwoWay时，改变值后会立即从源传播到目标，然而相反传递即==从目标到源——未必立即发生==**。当我们设置UpdateSourceTrigger时既可控制绑定中源的更新方式。

| 名称              | 说明                                                                                  |
| :-------------- | :---------------------------------------------------------------------------------- |
| PropertyChanged | 当目标属性发生改变时立即更新源                                                                     |
| LostFocus       | 当属性发生改变且目标丢失焦点时更新源                                                                  |
| Explicit        | 除非调用BindingExpression.UpdateSource()方法，否则无法更新源,获取绑定表达式对象参考[[第八章 元素绑定#3.3使用代码来检索绑定]] |
| Default         | 根据目标属性的元数据确定更新行为。大多数默认为PropertyChanged,但TextBox.Text属性默认行为为LostFocus                |
绑定延迟：Delay，等待数毫秒之后再提交更改。
绑定更新及其绑定延迟代码示例：
```XAML
 <TextBlock Margin="10" Text="Simple Text" Name="lblSampleText"
           FontSize="{Binding ElementName=sliderFontSize,Path=Value,Mode=TwoWay,UpdateSource=PropertyChanged,Delay=500}"/>
```
## 8.2绑定到非元素对象
**细节提示：WPF数据绑定基础结果无法获取私有信息或公有字段**
### 8.1Source属性
Source属性非常简单，唯一的问题就是为了进行数据绑定时需要提供数据对象。

1.在代码中创建一个静态对象，并使用该对象。或者，可使用来自.NET库类中的组件。
代码示例：
   ```Csharp
   <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=Source}"/>
   ```
2.绑定到先前作为资源的创建的对象。
代码示例：
```XAML
<Window.Resources>
    <FontFamily x:Key="CustomFont">Calibri</FontFamily>
</Window.Resources>
```

```XAML
<TextBlock Text="{Binding Source={StaticResource CustomFont}，Path=Source}"/>
```

### 8.2RelativeSource属性
通过RelativeSource属性可以根据相对对象的关系指向源对象。例如，可使用该属性将元素绑定到自身或其父元素。
代码示例：
```XAML
<TextBlock>
  <TextBlock.Text>
    <Binding Path="Title">
      <Binding.RelativeSource>
        <RelativeSource Mode="FindAncestor" AncestorType="{x:Type Window}"/>
      </Binding.RelativeSource>
    </Binding>
  </TextBlock.Text>
</TextBlock>
```
另一种常用的方法时使用Binding和RelativeSource标记扩展，将其合并为一个字符串。如下示例：
```XAML
<TextBlock Text="{Binding Path=Title,RelativeSource={RelativeSource FindAncestor,AncestorType={x:Type Window}}}"/>
```
创建RelativeSource对象时，FindAncestor模式有以下四种：

|       名称       | 说明                                                                           |
| :------------: | :--------------------------------------------------------------------------- |
|      Self      | 表达式绑定到同一元素的另一个属性上                                                            |
|  FindAncestor  | 表达式绑定到父元素。需要指定父元素AncestorType，此外还可以用AncestorLevel属性省略以指示希望查找到的父元素类型，该属性默认值为1 |
|  PreviousData  | 表达式绑定到数据绑定列表中的前一个数据项。在列表项中常用这种方法                                             |
| TemplateParent | 表达式绑定到应用模板的元素。只有当绑定位于控件模板或数据内部时，这种模式才工作                                      |
### 8.3DataContext属性
在某些情况下，会将大量元素绑定到同一个对象。
示例：

```XAML
<StackPanel>
  <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=LineSpacing}"/>
  <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=LineSpacing}"/>
  <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=LineSpacing}"/>
  <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=LineSpacing}"/>
  <TextBlock Text="{Binding Source={x:Static SystemFonts.IconFontFamily},Path=LineSpacing}"/>
</StacPanel>
```
解决以上大量重复的方法可以利用DataContext特性，例如：

```XAML

<StackPanel DataContext="{x:Static SysytemFonts.IconFontFamily}">
   <TextBlock Margin="5" Text="{Binding Path=Source}"/>
   ......
</StackPanel>
```
